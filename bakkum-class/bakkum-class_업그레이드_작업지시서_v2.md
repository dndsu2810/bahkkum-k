# 바꿈 수업관리(bakkum-class) 업그레이드 작업지시서 v2

**대상 앱:** bakkum-class (Cloudflare Workers)
**목적:** 단순 수정이 아니라, "있으면 더 좋은" 편의 기능 + 미적 완성도를 끌어올려 학원장이 매일 쓰고 싶은 앱으로 디벨롭한다.
**원칙:** 지금 토대(Pretendard 폰트 · 파란 포인트 · 16px 둥근 카드)는 좋으니 **갈아엎지 말고 그 위에 얹는다.**
**우선순위:** 🟦 편의기능(이번 핵심) · 🎨 디자인 미감 · 🔴 필수 개선(가독성·접근성)

---

## 📐 디자인 토큰 (현재 값 — 일관성 유지용)
실제 측정값이다. 새로 만들 때 이 값과 어긋나지 않게 한다.
- 폰트: `Pretendard Variable` (유지)
- 주 포인트색: `#2563EB`(blue-600), 진한 포인트: `#1D4ED8`(blue-700)
- 보조 포인트(중등 등): `#7C3AED`(violet-600)
- 카드: 배경 `#FFFFFF` · 모서리 `16px` · 테두리 `#EAEEF4` · 그림자 `rgba(15,23,42,0.05) 0 1px 2px`
- 본문 글씨(정상): `#475569`(slate-600)

### ➕ 새로 정의할 "상태 색" 팔레트 (지금 출결 버튼이 무채색이라 상태가 한눈에 안 옴)
| 의미 | 글자/아이콘 | 연한 배경 |
|---|---|---|
| 출석 · 완료 (성공) | `#16A34A` (green-600) | `#DCFCE7` |
| 지각 · 주의 | `#D97706` (amber-600) | `#FEF3C7` |
| 결석 · 지연 (위험) | `#DC2626` (red-600) | `#FEE2E2` |

> 규칙: **선택된 상태만 색으로 채우고**, 안 누른 버튼은 차분한 회색으로 둔다. 화면이 알록달록해지지 않게.

# 🟦 PART A. 편의 기능 추가 (이번 업그레이드의 핵심)

## A-1. 상단 학원명(로고) 클릭 → 홈으로 이동
- 좌측 상단 "바꿈영수학원" 영역을 누르면 **'오늘' 화면**으로 이동.
- 마우스 올리면 손가락 커서(`cursor:pointer`) + 살짝 강조, 키보드로도 선택·이동 가능하게(링크/버튼 역할 부여).
- **완료 기준:** 어느 화면에서든 로고를 누르면 오늘 화면으로 돌아온다.

## A-2. 상단 통계 카드 = 바로가기 버튼으로
오늘 화면의 5개 카드를 단순 숫자에서 **누르면 해당 화면으로 가는 바로가기**로 만든다.
| 카드 | 누르면 이동할 곳 |
|---|---|
| 오늘 수업 | 오늘 등원 학생 목록(같은 화면 아래로 스크롤) 또는 시간표 |
| 출결 미체크 | 출결 기록(오늘) · 미체크 학생부터 |
| 검사할 숙제 | 숙제 기록 |
| 보강 예정 | 보강 관리 |
| 오늘 테스트 | 테스트 기록 / 오늘 테스트 목록 |
- 카드에 hover 시 "누를 수 있다"는 신호(커서 + 미세하게 떠오름)를 준다.
- **완료 기준:** 각 카드를 누르면 의도한 화면으로 이동한다.

## A-3. 입력 후 즉각 피드백 + 실행취소
- 출결/숙제 처리하면 화면 우하단에 **토스트 알림**: 예) `최다연 · 출석 처리됨`.
- 토스트에 **실행취소(되돌리기)** 버튼을 짧게 노출(예: 5초). 잘못 누른 걸 바로 되돌릴 수 있게.
- "전체 출석" 같은 일괄 동작에도 동일하게 확인 토스트 + 되돌리기.
- **완료 기준:** 처리할 때마다 무엇이 됐는지 알려주고, 한 번의 클릭으로 되돌릴 수 있다.

## A-4. 날짜 이동 편하게
- 우상단 날짜 배지("2026년 6월 12일") 옆에 **‹ 어제 / 내일 ›** 화살표와 **"오늘로"** 버튼 추가.
- **완료 기준:** 화살표로 날짜를 하루씩 넘기고, 한 번에 오늘로 돌아올 수 있다.

## A-5. 학생 이름 클릭 → 학생 상세 (이미 학생 관리가 있다면 연결)
- 목록의 학생 이름을 누르면 그 학생의 출결·숙제·진도 이력으로 이동.
- **완료 기준:** 이름을 눌러 한 학생의 흐름을 바로 확인할 수 있다.

## A-6. 빈 상태 · 로딩 상태 안내
- 데이터 없을 때: "오늘 등원 예정 학생이 없어요" 처럼 **친절한 한 줄 + 다음 행동 제안**(예: 시간표 보기 버튼).
- 불러오는 중: 회색 자리표시(스켈레톤)로 "로딩 중"임을 보여줘 빈 화면 깜빡임 방지.
- **완료 기준:** 빈 화면·깜빡임 없이 항상 무슨 상태인지 보인다.

## A-7. (선택) 출결 입력 키보드 단축키 — 파워유저용
- 출결 입력에 포커스됐을 때 `1=출석 · 2=지각 · 3=결석`, `Enter=다음 학생`.
- **완료 기준:** 마우스 없이 키보드만으로 한 반 출결을 빠르게 끝낼 수 있다.

## A-8. (선택) 모바일 하단 탭바
- 좁은 화면에서 자주 쓰는 4~5개(오늘·시간표·출결·숙제)를 **하단 고정 탭바**로. 엄지로 닿는 위치.
- **완료 기준:** 한 손 모바일 사용 시 주요 화면을 엄지로 오갈 수 있다.

# 🎨 PART B. 디자인 미감 디벨롭

## B-1. 상태를 색으로 (PART 상단 팔레트 적용)
- 출석=초록 / 지각=앰버 / 결석=빨강으로, **선택된 상태만 채움**. 진도·숙제 완료/지연도 같은 규칙.
- **완료 기준:** 목록을 훑기만 해도 누가 출석·결석인지 색으로 바로 구분된다.

## B-2. 글자 위계(타이포 스케일) 정리
- 화면 제목 22~24px Bold / 섹션 제목 16~18px SemiBold / 본문 14px / 캡션 13px(`#64748B`).
- 대시보드 큰 숫자는 더 또렷하고 크게, 단위(명·건)는 작게 붙여 대비를 준다.
- **완료 기준:** 제목·본문·보조설명이 크기·굵기만으로 위계가 분명하다.

## B-3. 간격 리듬 통일 (8px 시스템)
- 여백을 4·8·12·16·24·32 단위로 통일. 카드 안쪽 패딩과 카드 사이 간격을 일정하게.
- **완료 기준:** 화면을 옮겨다녀도 간격이 들쭉날쭉하지 않다.

## B-4. 미세 인터랙션 (과하지 않게)
- 150~200ms 부드러운 전환. 카드 hover 시 살짝 떠오름(위로 2px + 그림자 약간 진하게), 버튼 누름 피드백, 탭 전환 부드럽게.
- **'동작 줄이기(reduce motion)' 설정을 켠 사용자에겐 애니메이션을 끈다.**
- **완료 기준:** 손이 닿는 곳이 살아있는 느낌이되, 산만하지 않다.

## B-5. 배지·태그 정리
- 학년 배지(초등/중등), 행사 태그(KTC수학경시대회 등)의 색·모양·크기를 한 세트로 통일.
- **완료 기준:** 태그들이 제각각이 아니라 한 디자인 언어로 보인다.

## B-6. ✨ 시그니처 요소 — "오늘 한 줄 브리핑"
- 오늘 화면 맨 위에, 그날을 한 문장으로 요약하는 **히어로 한 줄**:
  예) `오늘은 4명 등원 · 출결 4명 대기 · 숙제 4건 검사 · 테스트 5건`
- 숫자는 누르면 해당 위치로 이동(A-2와 연결). 톤은 ez쌤답게 따뜻하고 담백하게.
- 이게 이 앱만의 기억되는 포인트. 학원장이 아침에 딱 보면 하루가 잡히게.
- **완료 기준:** 오늘 화면을 열면 한 줄로 그날 할 일이 파악된다.

## B-7. (선택) 다크 모드
- Pretendard + slate 기반이라 무리 없음. 헤더나 설정에 토글 제공.
- **완료 기준:** 야간에도 눈부심 없이 쓸 수 있다.

## B-8. 마이크로카피 통일
- 버튼·토스트 표현을 한 흐름으로: 동작은 일어날 일 그대로("저장" → 토스트 "저장됨").
- 빈 상태/에러는 사과 대신 **무엇이 어떻게 되는지** 안내.
- **완료 기준:** 같은 동작이 화면마다 다른 말로 불리지 않는다.

# 🔴 PART C. 필수 개선 (가독성·접근성 — 빼먹지 말 것)

## C-1. 연한 회색 글씨 진하게 (최우선)
- 보조 설명글 현재 `#8A97A8`(명암비 2.97:1) → **최소 `#64748B`(약 4.75:1)**, 중요한 안내는 `#475569`.
- breadcrumb 현재 `#B8C1CF`(1.82:1) → **최소 `#64748B`**, 현재 위치 강조는 `#334155`.
- **완료 기준:** 모든 텍스트 명암비 4.5:1 이상. 밝은 창가에서도 또렷.

## C-2. 시간표 겹침 시 이름 잘림
- 한 수업 칸 너비가 약 88px 미만이면 개별 카드 대신 **요약 칩**(`정시우 외 2명` / `3건`)으로 묶고, 클릭 시 펼침.
- **완료 기준:** 3개 겹쳐도 누구인지/몇 건인지 확인 가능.

## C-3. 버튼 터치 영역 44px (태블릿·키오스크)
- 숙제 칩·완성도(0/50/80/100)·출결 버튼 등 터치 영역 최소 44×44px(padding으로 확장 가능), 칩 간격 확보.
- **완료 기준:** 태블릿에서 옆 버튼 오터치가 없다.

## C-4. 제목 구조 & 좁은 화면 로고
- 화면별 대표 제목을 `<h1>` 1개, 하위는 `<h2>/<h3>` 순서대로. 본문은 `<main>`으로 감싼다.
- 좁은 화면 모드에서 로고/브랜드는 가로 스크롤 탭과 분리해 **고정(sticky)**.
- **완료 기준:** 화면별 h1 존재, 좁은 화면에서 로고가 사라지지 않음.

## C-5. 시간표 빈 시간 정리
- 진입 시 첫 수업 시간 근처로 자동 스크롤하거나 비어 있는 앞쪽 시간 행을 접는다.
- **완료 기준:** 시간표를 열면 그 주 실제 수업이 바로 보인다.

## ✅ 유지할 점 (건드리지 말 것)
- 데스크탑 사이드바 그룹 분류(메인/수업 기록/관리), 카드 레이아웃, 파란 포인트 일관성.
- Pretendard 폰트, 키보드 포커스 표시, `lang="ko"`, `<nav>` 랜드마크, 시간표 오늘 컬럼 강조 + 색 범례.

## 작업 순서 제안
1. **C-1(글씨 대비)** 먼저 — 체감 가장 큼.
2. **A-1·A-2(로고/카드 바로가기)** — 손이 자주 가는 편의.
3. **B-1·B-6(상태 색 + 오늘 한 줄 브리핑)** — 미감 핵심.
4. 나머지 편의·미감 → 필수 개선 마무리.
5. 매 단계 **기존 기능(출결·숙제·Notion 연동) 회귀 확인.**
