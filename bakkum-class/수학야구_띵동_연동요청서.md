# 수학 야구(수학 전광판) — 구성 정리 & 띵동 키오스크 연동 요청서

> 작성: 쏘이지(바꿈영수학원 앱) / 대상: 수학 키오스크 **띵동**
> 목적: 띵동 화면에서도 학생의 수학 야구 점수(스트라이크·볼·아웃·회차)를 **그대로 읽어 보여주기** 위함.
> 핵심 원칙: **점수 계산은 쏘이지가 다 끝내서 내려준다. 띵동은 받아서 그리기만 하면 된다.**

---

## 1. 수학 야구가 뭔가요 (개념)

학생의 수업 태도를 야구 규칙으로 게임화한 점수판입니다.

| 항목 | 뜻 | 어떻게 쌓이나 |
|---|---|---|
| **스트라이크(S)** | 벌점 | 출결·숙제에서 **자동**으로 쌓임 |
| **볼(B)** | 상점 | 선생님이 좋은 행동에 **직접** 줌 |
| **아웃(O)** | 누적 경고 | 스트라이크 3개가 모이면 아웃 1개 |
| **회차** | 지금 몇 번째 판인지 | 1회부터 시작, 쓰리아웃 되면 2회·3회…로 |

**규칙 흐름**
- 스트라이크 **3개** → 아웃 **1개** (스트라이크는 0으로)
- 볼 **4개**(볼넷) → 아웃 **1개 감소** (아웃 없으면 스트라이크 1개 감소, 둘 다 없으면 꿀로 전환)
- 아웃 **3개**(쓰리아웃) → **보충 대상** + 점수 초기화 + **회차 +1**
- 매월 1일 S·B·O 초기화 (단, **회차·보충대기·꿀은 유지**)
- 볼은 하루 최대 2개까지 적립

**자동 스트라이크가 붙는 경우(기본값, 각 1개)**
- 출결: 무단결석 / 지각 / 수업태도 미흡
- 숙제: 숙제 지연 / 숙제 50% 이하
- ※ **한 기록당 스트라이크는 최대 1개** (지연이면서 50%이하인 숙제도 1개만)

> 위 기준값·항목은 쏘이지 선생님 화면(수학 야구 → 벌·상 항목 설정)에서 **추가·수정·삭제** 가능합니다. 그래서 띵동은 값을 하드코딩하지 말고 **API가 주는 숫자를 그대로** 쓰면 됩니다.

## 2. 띵동이 호출할 API (요청 사항)

### 학생 전광판 조회
```
GET /api/baseball/board?student_id=<로스터ID>
```
- 인증: 쏘이지 세션 쿠키. (키오스크 전용 호출이 필요하면 별도 협의 — 아래 4번)
- `student_id` = 학생 로스터 ID(숫자). 띵동이 등하원 때 쓰는 출석번호/온라인ID와 매핑 필요(쏘이지 `class_student_meta`에 매핑 있음).
- 학생 본인 세션이면 `student_id` 없이 호출 시 본인 것 반환.

**응답(JSON)**
```jsonc
{
  "board": {
    "studentId": "123",
    "S": 2, "B": 1, "O": 0,        // 스트라이크·볼·아웃 현재 개수
    "penaltyRounds": 1,             // 누적 보충(쓰리아웃 초기화) 횟수
    "pendingMakeup": false,         // 지금 보충 대상인지
    "honey": 0,                     // 볼4개가 꿀로 전환된 누적
    "status": "warn",              // clean | good | warn | makeup
    "monthLabel": "2026-06",
    "goal": "스트라이크 하나만 더 쌓이면 아웃이에요. 조심!",
    "recent": [ /* 최근 8건 */ ],
    "history": [ /* 전체(최신순, 최대 120건) */ ]
  },
  "photo": "https://.../up/xxxx.jpg"  // 학생 사진(없으면 "")
}
```
- **현재 회차 = `penaltyRounds + 1`** (1부터 시작).
- 수학 수강생이 아니면 `board`가 `null`. 그러면 띵동도 전광판을 숨기면 됩니다.

**history / recent 한 건의 형태**
```jsonc
{
  "date": "2026-06-22",
  "tone": "strike",   // strike | ball | minus | makeup | honey
  "label": "숙제 지연",
  "delta": "스트라이크 +1",
  "round": 1           // 이 기록이 속한 회차
}
```
→ `history`를 `round`로 묶으면 "회차별 기록 보기"를 그대로 만들 수 있습니다.

## 3. 띵동에서 보여줄 화면 (제안)

쏘이지 학생 화면과 톤을 맞추면 좋습니다.

- **전광판 패널**(어두운 배경 `#3A2A1A`)에 LED 점 3줄
  - 스트라이크(주황 `#FF7A3D`) 3칸 · 볼(초록 `#34C759`) 4칸 · 아웃(빨강 `#FF4438`) 3칸 · 꺼진 칸 `#5A4630`
  - 각 줄 끝에 **개수 숫자**(예: `2개`)
- 상단에 **현재 N회**(회차), 보충 대상이면 빨강 "보충 대상" 뱃지
- 하단에 `스트라이크 N개 · 볼 N개 · 아웃 N개` 요약 + `goal` 안내문
- (선택) **회차별 기록 보기** — `history`를 round로 묶어 표시

> 색·문구는 위 응답 값을 그대로 쓰면 쏘이지와 100% 일치합니다.

## 4. 협의가 필요한 부분

1. **인증**: 띵동(키오스크)은 보통 무인증/공용 단말입니다. 학생 점수는 개인정보라 현재는 로그인 세션이 필요합니다.
   - 제안 A: 키오스크 전용 읽기 토큰(헤더 키) 발급 → `GET /api/baseball/board`만 허용.
   - 제안 B: 등하원 펀치(체크인) 직후 그 학생 board를 함께 내려주기.
   - → 어떤 방식이 띵동에 편한지 알려주세요.
2. **학생 ID 매핑**: 띵동 출석번호 ↔ 쏘이지 로스터ID 매핑 확인(쏘이지 `class_student_meta.checkin_no` / `online_id` 보유).
3. **갱신 주기**: 쏘이지 학생앱은 15초 폴링. 띵동은 등하원 이벤트 때 1회 조회면 충분할 듯.

## 6. (추가) 상벌점 항목 안내 — 띵동에도 표시 요청

쏘이지 학생 화면에는 "상벌점 항목 보기"가 있어요(무슨 행동이 스트라이크/볼 몇 개인지). **띵동에도 똑같이 보여주고 싶습니다.**

**쏘이지 쪽은 작업 완료** — board API 응답에 이미 `rules`(켜진 항목만)와 `cfg`(기준값)가 들어갑니다. **추가 API·토큰 변경 없음.** 띵동은 받은 값만 렌더하면 됩니다.

응답 추가 필드:
```jsonc
{
  // ...board, photo...
  "rules": [
    { "kind": "strike", "label": "무단결석", "points": 1, "sort": 1 },   // 벌점(스트라이크)
    { "kind": "ball",   "label": "자진등원", "points": 1, "sort": 2 }     // 상점(볼)
    // enabled=true 인 항목만 내려갑니다(끈 항목은 제외)
  ],
  "cfg": { "strikesPerOut": 3, "ballsToClearOut": 4, "outsForMakeup": 3, "dailyBallCap": 2, "monthlyReset": true }
}
```

**표시 가이드(쏘이지와 동일하게):**
- `rules`를 `kind`로 나눠 두 칸: **벌점·스트라이크**(`kind:"strike"`) / **상점·볼**(`kind:"ball"`), 각 `sort` 오름차순.
- 한 줄 = `label`  ·  `+points`
- 아래 안내문 한 줄(`cfg`로 생성):
  `스트라이크 {strikesPerOut}개 = 아웃 1개 · 볼 {ballsToClearOut}개 = 아웃 1개 감소 · 아웃 {outsForMakeup}개 = 보충{ monthlyReset이면 " · 매월 초기화"}`
- 누르면 펼치고 다시 누르면 닫히는 토글이면 좋아요(쏘이지와 동일 UX).

**자동 반영**: 선생님이 항목을 바꾸면 DB에 바로 저장돼요. 띵동은 board를 다시 부를 때(등하원 펀치마다, 또는 폴링) 최신 `rules`를 받으므로 **별도 동기화 불필요**. 캐싱한다면 board 재조회 시 같이 갱신만 해주세요.

## 7. (추가) 전광판 표시 디테일 — 쏘이지 최신 UI와 맞추기

쏘이지 학생 전광판을 아래처럼 다듬었어요. 띵동도 같은 느낌으로 맞춰주세요. **API 변경 없음** — 모두 기존 `board` 값으로 표시됩니다.

1. **회차(1회·2회…)는 학생 이름 옆에** 작은 칩으로.
   - 값 = `board.penaltyRounds + 1` (1부터 시작, 쓰리아웃마다 +1).
   - 예: `민서준 [1회]`
2. **"좋아요/주의" 같은 상태 칩은 표시하지 않음** (없앴어요).
   - 단, **`board.pendingMakeup === true`** 이면 빨강 **"보충 대상"** 표시는 유지.
   - (참고: `board.status`는 안 써도 됩니다. 쓰면 `pendingMakeup`만 강조용으로.)
3. 점수 요약 한 줄은 유지: `스트라이크 {S}개 · 볼 {B}개 · 아웃 {O}개` + `board.goal` 안내문.

> 정리: 회차 = 이름 옆 칩 / 좋아요·주의 칩 제거 / 보충 대상일 때만 빨강 강조.
