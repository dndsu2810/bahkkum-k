# 바꿈 수업관리 앱 — "등하원" 카테고리 추가 지시서

> 수업관리 앱에 등하원 기능을 추가하는 작업 지시서입니다.
> 확정된 결정사항을 그대로 반영했고, 이 문서만 보고 만들 수 있게 정리했습니다.

## 1. 한 줄 요약

수업관리 앱 사이드 메뉴에 "등하원" 카테고리를 새로 만든다. 학생이 등하원을 찍으면, 선생님이 확인한 뒤 솔라피 알림톡으로 학부모에게 알림이 간다. 수학 키오스크와는 연결하지 않는다.

## 2. 꼭 지킬 원칙 (가볍게 유지)

- 학생·학부모 번호는 **기존 학생 데이터를 그대로 재사용**한다. 명부를 새로 만들거나 복사하지 않는다. (학부모 번호 칸이 없으면 기존 학생 데이터에 칸만 추가)
- 등하원 기록 화면은 **날짜별 접기(collapsible)**로 만들어, 데이터가 쌓여도 화면이 길어지지 않게 한다. (수업관리에서 쓰던 방식 재사용)
- 기존 수업관리(숙제·진도·시험) 화면은 건드리지 않는다. 새 카테고리만 추가한다.

## 3. 확정된 결정사항

- 사이드 메뉴에 **"등하원" 공통 카테고리를 신규 추가**한다. 관리·발송은 모두 여기서 한다.
- **학생 상세 화면**에는 그 학생의 **등하원 이력 "조회용" 섹션만** 추가한다. (여기서는 수정·발송 기능 없음)
- 등하원 구분은 **학생이 "등원" / "하원" 버튼을 직접 선택**한다. (시간대·횟수 기반 자동 판정 안 함)
- 알림톡 발송은 **선생님이 "알림 보내기" 버튼을 누를 때만** 나간다. (시간을 수정한다고 자동으로 발송되지 않음)
- 이미 한 번 보낸 학생에게 다시 보낼 때는, 시스템이 자동으로 **"정정 알림"** 양식으로 전환해 보낸다.
- **키오스크와는 연결하지 않는다.** (완전히 별개)

## 4. 만들 화면 3개

### (가) 학생용 — 등하원 찍기 (태블릿)

- 학생이 본인을 찾아 선택한다. (반별 또는 검색)
- 큼직한 "등원" / "하원" 버튼 두 개.
- 누르면 현재 시간이 자동 기록되고, 화면에 "○○ 등원 완료" 같은 확인 표시가 뜬다.
- 아이들이 쉽게 쓰도록 버튼과 글씨를 크고 단순하게.

### (나) 선생님용 — 등하원 관리 (메인, 오늘 중심)

- 상단: 오늘 날짜 + 등원/하원/미발송 요약 숫자.
- 오늘 전체 학생 목록. 학생마다 다음을 표시한다.
  - 상태: 등원완료 / 하원완료 / **미발송(빨간색 강조)** / 정정됨
  - 찍힌 시간 + 시간 수정 기능
  - "학부모에게 알림 보내기" 버튼
- 안 찍고 간 학생(미발송)을 한눈에 알아보게 한다.
- 지난 기록은 날짜별 접기로 정리한다.

### (다) 학생 상세 — 등하원 이력 (조회용)

- 그 학생의 등하원 기록을 날짜순으로 보여준다. (읽기만, 수정·발송 없음)
- 학부모 상담 때 보여주는 용도.

## 5. 저장할 데이터 (등하원 기록)

- 학생, 날짜, 구분(등원/하원), 시간, 발송 여부, 발송 시각, 정정 여부.
- 학생-학부모 번호는 **기존 학생 데이터에서 가져온다.** (등하원 쪽에 따로 저장하지 않는다)

## 6. 알림톡(솔라피) 연동

- 솔라피를 통해 발송한다. 템플릿은 3종이 필요하다: **등원 / 하원 / 정정**.
- 발송할 때마다 발송 기록을 남긴다. (누구에게, 언제, 성공/실패)
- 솔라피 비밀키(API 키)는 코드에 직접 적지 말고 안전한 곳에 따로 보관한다.

## 7. 안전장치

- **테스트 모드**: 실제 학부모가 아니라 지정한 본인 번호로만 발송되게 하는 스위치. 처음에는 반드시 이 모드로 충분히 시험한다.
- **하루 발송 한도**: 정해둔 건수를 넘으면 멈추고 알린다. (버그로 인한 반복 발송·비용 폭주 방지)

## 8. 이번 범위에서 제외 (다음 단계로)

- 등하원 자동 판정
- 학부모 앱 푸시 알림
- 양방향(학부모 답장)
- 발송 실패 시 문자 자동 대체

## 9. 만드는 순서 (작게 쪼개서 진행)

1. 사이드 메뉴 "등하원" + 빈 관리 화면 골격
2. 학생용 찍기 화면 → 기록 저장
3. 선생님 관리 화면: 상태 표시 + 시간 수정
4. 솔라피 발송(테스트 모드) → 정정 발송 자동 구분
5. 학생 상세에 이력 조회 섹션 추가

각 단계가 끝날 때마다 동작을 확인하고 다음으로 넘어간다.
