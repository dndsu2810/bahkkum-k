| 문서 버전 | 작성일 | 개발 대상 | 화면 수 |
|---|---|---|---|
| 통합본 v1.0 | 2026-06-17 | Claude Code | 4종 |

> 기존 「등하원 업데이트 지시서」와 「디자인 시안 요청서」, 그리고 와이어프레임에서 추가된 **수업시간 리포트** 화면까지 — 지금까지 확정·설계된 내용을 한 문서로 통합했습니다. 이 문서 하나만 보고 작업을 시작할 수 있도록 정리했습니다.

---

## 한 줄 요약

수업관리 앱 사이드 메뉴에 **"등하원" 공통 카테고리**를 새로 만든다. 학생이 등하원을 직접 찍으면, 선생님이 확인한 뒤 **솔라피 알림톡**으로 학부모에게 알림이 나간다.

---

## 1. 꼭 지킬 원칙

- 학생·학부모 번호는 **기존 학생 데이터를 그대로 재사용**한다. 명부를 새로 만들거나 복사하지 않는다. (학부모 번호 칸이 없으면 기존 학생 데이터에 칸만 추가)
- 등하원 기록 화면은 **날짜별 접기(collapsible)**로 만들어, 데이터가 쌓여도 화면이 길어지지 않게 한다. (수업관리에서 쓰던 방식 재사용)
- 기존 수업관리(숙제·진도·시험) 화면은 **건드리지 않는다.** 새 카테고리만 추가한다.

---

## 2. 확정된 결정사항

- 사이드 메뉴에 **"등하원" 공통 카테고리를 신규 추가**한다. 관리·발송은 모두 여기서 한다.
- **학생 상세 화면**에는 그 학생의 **등하원 이력 "조회용" 섹션만** 추가한다. (수정·발송 기능 없음)
- 등하원 구분은 **학생이 "등원"/"하원" 버튼을 직접 선택**한다. (시간대·횟수 기반 자동 판정 안 함)
- 알림톡 발송은 **선생님이 "알림 보내기" 버튼을 누를 때만** 나간다. (시간 수정만으로 자동 발송되지 않음)
- 이미 한 번 보낸 학생에게 다시 보낼 때는, 시스템이 자동으로 **"정정 알림"** 양식으로 전환해 보낸다.
- **키오스크와는 연결하지 않는다.** (수학 전용 키오스크와는 완전히 별개)

---

## 3. 만들 화면 4종

> (가)~(다)는 기존 지시서의 3종, (라)는 와이어프레임에서 새로 제안된 화면입니다. 레이아웃 시안은 **「등하원 와이어프레임」** 파일을 참조하세요.

### (가) 학생용 — 등하원 찍기 · 태블릿

- 학생이 본인을 찾아 선택한다. **출석번호 입력(키패드)** 또는 반별/검색. 없는 번호면 오류 안내.
- 큼직한 **"등원" / "하원" 버튼**. 영어·수학 동시 수강생은 **과목 × 등하원 4버튼**, 단일 과목은 2버튼.
- 누르면 현재 시간이 자동 기록되고, **"○○ 등원 완료"** 같은 큰 확인 표시가 뜬다.
- 아이들이 쉽게 쓰도록 버튼·글씨를 크고 단순하게. 터치 영역 넉넉히.

### (나) 선생님용 — 등하원 관리 · 메인(가장 중요) · PC/태블릿

- 매일 여러 번 빠르게 훑는 화면 → **한눈에 스캔**이 1순위.
- 상단: 오늘 날짜 + **등원/하원/미발송 요약 숫자**.
- 오늘 전체 학생 목록(카드형 또는 표·행형). 항목마다 — 이름, **상태 배지**(등원완료/하원완료/**미발송**/정정됨), 찍힌 시간 + **시간 수정**, **"알림 보내기" 버튼**.
- **미발송은 빨간색**으로 확 눈에 띄게 (테두리 + 배지). 발송 완료 행은 버튼 대신 "발송됨 ✓"로 중복 발송 방지.
- 지난 기록은 날짜별 접기로 정리.

### (다) 학생 상세 — 등하원 이력 · 조회용 · 기존 수업관리 환경

- 그 학생의 등하원 기록을 날짜순으로 보여준다. **읽기 전용** — 수정·발송 없음.
- 표현은 **타임라인** 또는 **표**. 학부모 상담 때 보여줄 용도라 정돈된 느낌.
- 학생 상세 화면 안의 한 섹션(숙제·진도와 나란히 "등하원" 탭).

### (라) 학생별 수업시간 리포트 · 신규 제안 · 상담/청구용

> 와이어프레임 검토 중 추가된 화면. 등하원 기록을 **월 단위로 합산**해 등록 수업시간 대비 초과/부족을 보여줘, 수업 추가·청구 상담에 바로 쓰는 용도. **구현 여부는 검토 후 결정**합니다.

- 상단 4개 숫자: **이번 달 누적 / 등록 대비(초과·부족) / 지난달 대비 / 남은 수업**.
- 주차별 막대그래프 + 등록 대비 게이지(초과분은 **빨강**) + 지난달 비교.
- 일별 기록은 길어지지 않게 **월별 접기**로 정리. 상담 메모 한 줄 포함.

## 4. 저장할 데이터 (등하원 기록)

| 항목 | 설명 |
|---|---|
| 학생 | 기존 학생 데이터 참조 |
| 날짜 | 기록일 |
| 구분 | 등원 / 하원 (+ 과목) |
| 시간 | 찍힌 시각 (수정 가능) |
| 발송 여부 / 발송 시각 | 알림톡 발송 상태 + 발송된 시각 |
| 정정 여부 | 정정 발송으로 전환되었는지 |

- 학생–학부모 번호는 **기존 학생 데이터에서 가져온다.** 등하원 쪽에 따로 저장하지 않는다.

---

## 5. 알림톡(솔라피) 연동

- 솔라피를 통해 발송한다. 템플릿은 **3종** 필요: **등원 / 하원 / 정정**. *(문구는 작성·심사 예정)*
- 발송할 때마다 발송 기록을 남긴다. **(누구에게, 언제, 성공/실패)**
- 솔라피 비밀키(API 키)는 **코드에 직접 적지 말고** 안전한 곳에 따로 보관한다.

| 템플릿 | 내용 |
|---|---|
| 등원 | 학생명 · 등원 시각 안내 |
| 하원 | 학생명 · 하원 시각 안내 |
| 정정 | 앞서 보낸 알림의 정정 안내 |

---

## 6. 안전장치

- **테스트 모드** — 실제 학부모가 아니라 지정한 본인 번호로만 발송되게 하는 스위치. 처음에는 반드시 이 모드로 충분히 시험한다.
- **하루 발송 한도** — 정해둔 건수를 넘으면 멈추고 알린다. (버그로 인한 반복 발송·비용 폭주 방지)

---

## 7. 이번 범위에서 제외 (다음 단계로)

- 등하원 자동 판정
- 학부모 앱 푸시 알림
- 양방향(학부모 답장)
- 발송 실패 시 문자 자동 대체

---

## 8. 만드는 순서

> 작게 쪼개서 진행 — 각 단계가 끝날 때마다 동작을 확인하고 다음으로 넘어간다.

1. 사이드 메뉴 "등하원" + 빈 관리 화면 골격
2. 학생용 찍기 화면 → 기록 저장
3. 선생님 관리 화면: 상태 표시 + 시간 수정
4. 솔라피 발송(테스트 모드) → 정정 발송 자동 구분
5. 학생 상세에 이력 조회 섹션 추가
6. *(검토 후)* 학생별 수업시간 리포트 추가
