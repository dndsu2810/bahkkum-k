# 바꿈 수업관리 — 학생 메시지(알림) 기능 추가 지시서

> 대상: `bakkum-class.dndsu2810.workers.dev` (GitHub: `dndsu2810/bahkkum`)
> 성격: **새 기능 추가.** 선생님 → 학생 단방향 알림 + 학생의 1회 답장.

---

## 0. 한 줄 요약

> **선생님이 학생에게 메시지를 보내면, 학생 화면 종 아이콘에 알림이 뜨고 → 학생이 열어 읽고 → 답장은 딱 한 번만. 채팅 아님.**

---

## 1. 기능 개요

- **방향:** 선생님 → 학생. (학생이 먼저 보낼 수는 없음)
- **답장:** 학생은 받은 메시지 1건당 **답장 1회만** 가능. 주고받는 채팅 아님.
- **알림:** 새 메시지가 오면 잠깐 강조 표시가 떴다가, **읽거나 답장하면 사라짐.** (계속 떠 있지 않음)
- **기록:** 메시지는 **날짜별로 쌓여** 학생도 선생님도 지난 내역을 볼 수 있음.

---

## 2. 데이터 — 어디에, 어떻게 저장

- 메시지는 **앱 자체 저장소(D1)에 새 표(messages)를 만들어 저장**한다. 노션은 사용하지 않는다.
- **단체·다중 발송이라도, 보낸 대상 학생 수만큼 메시지를 개별로 만든다.** (학생별 읽음·답장 추적을 위해)

메시지 1건에 담길 정보(예시):

| 항목 | 내용 |
|------|------|
| 보낸 사람 | 초등수학 담당 / 원장 (이름 표시) |
| 받는 학생 | 학생 식별값 |
| 내용 | 메시지 본문 |
| 보낸 시각 | 날짜·시간 |
| 읽음 여부 / 읽은 시각 | 안읽음 → 읽음 |
| 답장 내용 / 답장 시각 | 답장 1회 (없을 수도) |

---

## 3. 보내는 쪽 — 선생님/원장 화면

### 3-1. 권한 (누가 보낼 수 있나)
- **초등수학 담당(지현)** 과 **원장님** 만 발송 가능.
- 그 외 강사에게는 이 발송 메뉴가 **아예 보이지 않게** 한다.

### 3-2. 진입 경로
- **원장님:** "**학생에게 메시지 보내기**" 카테고리(메뉴)를 새로 추가.
- **지현(초등수학):** **수학 수업관리** 안에 발송 메뉴를 추가.
- (두 경로의 발송 화면은 같은 모양으로 통일)

### 3-3. 발송 화면 구성
1. **대상 선택** — 개별 페이지가 있는 학생들 중에서 고른다.
   - **단체 발송** (후보 전체에게 한 번에)
   - **선택 발송** (체크박스로 여러 명 골라서)
2. **메시지 작성** — 본문 입력칸 1개. (간단하게)
3. **보내기** 버튼. 보낸 뒤 "○명에게 보냈어요" 확인.

### 3-4. 보낸 기록 확인
- 보낸 메시지를 날짜별로 보고, **누가 읽었는지 / 답장이 왔는지** 한눈에 확인.
- 학생 답장이 오면 그 자리에서 확인. (선생님은 다시 답장하지 않음 — 1회성)

---

## 4. 받는 쪽 — 학생 화면

### 4-1. 종 아이콘 + 알림 배지
- 화면 상단에 **종 아이콘**. 안 읽은 메시지 수를 **숫자 배지**로 표시.

### 4-2. 새 메시지 강조 (일시적)
- 새 메시지가 도착하면 **"선생님이 메시지를 보냈어요"** 강조 말풍선/팝업이 잠깐 뜬다.
- 계속 떠 있지 않는다. 잠시 후 사라지되, **종 위 알림 점·숫자**로는 남겨 둔다.
- **읽거나 답장하면** 강조 표시와 배지가 사라진다(갱신).

### 4-3. 메시지함 (종을 누르면)
- 받은 메시지가 **날짜별로 묶여** 목록으로 보인다. 안 읽은 건 굵게/점으로 강조.
- 메시지를 열면: **보낸 사람 · 내용 · 받은 날짜**가 보이고
  - **[읽음 확인]** 또는 **[답장 보내기]** 가능.

### 4-4. 답장 (딱 한 번)
- 짧은 입력칸 + 보내기. **한 번 보내면 입력칸은 잠기고 "답장함" 표시.**
- 추가 답장·수정 불가. (채팅처럼 계속 주고받지 않음)

---

## 5. UI/UX 원칙 (깔끔하게, 핵심만)

- 직전 통일 작업의 **공통 부품·토큰을 그대로 사용**한다. (새 디자인 만들지 말 것)
- 화면 요소는 최소로: 종 / 배지 / 메시지 카드 / 답장칸. 군더더기 없이.
- 글자 대비·터치 영역 등 접근성 기준 유지.
- 강조 팝업은 **방해되지 않게** — 짧게 뜨고, 닫기 쉽게.

---

## 6. ⚠️ 주의사항

1. **노션 동기화는 건드리지 않는다.** 메시지는 앱 저장소(D1)에만 쌓인다.
2. **답장은 1회.** 무한히 주고받는 채팅이 되지 않게 막는다.
3. **권한 없는 강사에게는 발송 메뉴를 숨긴다.**
4. 학생 화면 종/팝업이 **기존 화면을 느리게 만들지 않게** — 가볍게 확인하고, 깜빡임 없이.

---

## 7. 작업 순서

```
1) 앱 저장소(D1)에 메시지 표(messages) 추가
2) 발송 화면 (권한 체크 → 대상 선택[단체/다중] → 작성 → 발송)
3) 발송 진입 메뉴 추가 (원장: '학생에게 메시지 보내기' / 지현: 수학 수업관리 안)
4) 학생 화면 수신 (종 + 배지 + 일시 강조 팝업)
5) 메시지함 (날짜별 목록) + 읽음 처리
6) 답장 1회 + 발송 기록에서 답장 확인
7) 전체 점검 (권한·읽음·답장·날짜기록 동작 확인)
```
