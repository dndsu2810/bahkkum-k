## 개요

쏘이지 앱에 **회의록 관리** 기능을 새로 추가합니다.  
음성 파일을 업로드하면 → OpenAI Whisper로 텍스트 변환 → Claude API로 요약 → D1에 저장.  
기존 코드, 데이터, Notion 연동 로직은 절대 건드리지 않습니다.

---

## 사전 확인 사항

아래 두 환경변수가 Cloudflare Workers에 등록되어 있어야 합니다.  
없다면 원장님께 값을 받아서 `wrangler secret put` 또는 Cloudflare Dashboard → Workers & Pages → bakkum → Settings → Variables 에서 추가하세요.

- `OPENAI_API_KEY` — OpenAI에서 발급한 API 키 (Whisper 음성인식용)
- `ANTHROPIC_API_KEY` — Anthropic에서 발급한 API 키 (Claude 요약용)

---

## 1단계 — D1 데이터베이스 테이블 추가

D1 콘솔 또는 `wrangler d1 execute`로 아래 SQL을 실행합니다.  
**기존 테이블은 절대 수정하지 않습니다.**

```sql
CREATE TABLE IF NOT EXISTS meeting_records (
  id          INTEGER PRIMARY KEY AUTOINCREMENT,
  title       TEXT NOT NULL,
  meeting_date TEXT NOT NULL,
  attendees   TEXT DEFAULT '',
  raw_text    TEXT DEFAULT '',
  summary     TEXT DEFAULT '',
  created_at  TEXT DEFAULT (datetime('now', 'localtime'))
);
```

## 2단계 — Worker 백엔드 라우트 추가

기존 Worker 라우팅 처리 블록 안에 아래 라우트들을 추가합니다.  
기존 라우트는 절대 수정하지 않고, 신규 경로만 삽입합니다.

```javascript
// ─── 회의록: 목록 조회 ─────────────────────────────────
if (path === '/api/meetings' && method === 'GET') {
  const { results } = await env.DB.prepare(
    `SELECT id, title, meeting_date, attendees, created_at
     FROM meeting_records
     ORDER BY meeting_date DESC, created_at DESC`
  ).all();
  return jsonResponse(results);
}

// ─── 회의록: 상세 조회 ────────────────────────────────────
if (path.match(/^\/api\/meetings\/\d+$/) && method === 'GET') {
  const id = path.split('/').pop();
  const meeting = await env.DB.prepare(
    'SELECT * FROM meeting_records WHERE id = ?'
  ).bind(id).first();
  if (!meeting) return new Response('Not found', { status: 404 });
  return jsonResponse(meeting);
}

// ─── 회의록: 음성 변환 + AI 요약 ─────────────────────────
if (path === '/api/meetings/transcribe' && method === 'POST') {
  try {
    const formData = await request.formData();
    const audioFile  = formData.get('audio');   // 음성 파일 (선택)
    const manualText = formData.get('text');    // 직접 입력 텍스트 (선택)

    let rawText = '';

    if (audioFile && audioFile.size > 0) {
      // ① Whisper로 음성 → 텍스트
      const wForm = new FormData();
      wForm.append('file', audioFile);
      wForm.append('model', 'whisper-1');
      wForm.append('language', 'ko');

      const wRes = await fetch('https://api.openai.com/v1/audio/transcriptions', {
        method: 'POST',
        headers: { 'Authorization': `Bearer ${env.OPENAI_API_KEY}` },
        body: wForm,
      });

      if (!wRes.ok) {
        const errText = await wRes.text();
        return jsonResponse({ error: '음성 변환 실패: ' + errText }, 500);
      }
      const wData = await wRes.json();
      rawText = wData.text;

    } else if (manualText) {
      rawText = manualText;
    } else {
      return jsonResponse({ error: '음성 파일 또는 텍스트를 입력해주세요.' }, 400);
    }

    // ② Claude로 요약
    const cRes = await fetch('https://api.anthropic.com/v1/messages', {
      method: 'POST',
      headers: {
        'x-api-key': env.ANTHROPIC_API_KEY,
        'anthropic-version': '2023-06-01',
        'content-type': 'application/json',
      },
      body: JSON.stringify({
        model: 'claude-sonnet-4-6',
        max_tokens: 2048,
        messages: [{
          role: 'user',
          content: `다음은 학원 회의 내용입니다. 아래 형식으로 정확하게 요약해주세요.
각 항목은 반드시 "- " 불릿으로 작성하고, 내용이 없으면 "- 해당 없음"이라고 써주세요.

## 핵심 내용
(회의에서 논의된 주요 내용 3~5가지)

## 결정사항
(회의에서 확정된 사항들)

## 할일 목록
(담당자 포함, 예: - [지현] 교재 주문 확인)

---
회의 내용:
${rawText}`,
        }],
      }),
    });

    if (!cRes.ok) {
      return jsonResponse({ error: 'AI 요약 생성 실패' }, 500);
    }
    const cData = await cRes.json();
    const summary = cData.content[0].text;

    return jsonResponse({ rawText, summary });

  } catch (e) {
    return jsonResponse({ error: e.message }, 500);
  }
}

// ─── 회의록: 저장 ─────────────────────────────────────────
if (path === '/api/meetings' && method === 'POST') {
  const { title, meeting_date, attendees, raw_text, summary } = await request.json();
  if (!title || !meeting_date) {
    return jsonResponse({ error: '제목과 날짜는 필수입니다.' }, 400);
  }
  const result = await env.DB.prepare(
    `INSERT INTO meeting_records (title, meeting_date, attendees, raw_text, summary)
     VALUES (?, ?, ?, ?, ?)`
  ).bind(title, meeting_date, attendees || '', raw_text || '', summary || '').run();
  return jsonResponse({ id: result.meta.last_row_id, success: true });
}

// ─── 회의록: 삭제 ─────────────────────────────────────────
if (path.match(/^\/api\/meetings\/\d+$/) && method === 'DELETE') {
  const id = path.split('/').pop();
  await env.DB.prepare('DELETE FROM meeting_records WHERE id = ?').bind(id).run();
  return jsonResponse({ success: true });
}
```

> jsonResponse 헬퍼가 없다면 기존 코드의 JSON 응답 패턴에 맞춰 교체하세요.

## 3단계 — 프론트엔드: 메뉴에 "회의록" 탭 추가

화면: 하단 탭 내비게이션 (또는 사이드 메뉴)  
지금 모습: 회의록 탭 없음  
바꿀 동작: 기존 탭 맨 끝에 "회의록" 탭 추가  
아이콘: SVG 문서 + 마이크 조합 아이콘 (이모지 금지)  
클릭 시: /meetings 경로로 이동  
다 됐다고 볼 기준: 탭바에 "회의록" 항목이 보이고 클릭하면 목록 화면으로 이동

## 4단계 — 프론트엔드: 회의록 목록 화면

화면: /meetings  
지금 모습: 없음 (신규 생성)  
바꿀 동작:

- GET /api/meetings 호출해서 목록 렌더링
- 각 항목 카드에 표시: 회의 제목 / 날짜 / 참석자 (쉼표 구분)
- 카드 클릭 시 상세 화면으로 이동
- 우상단에 "새 회의록" 버튼 → 작성 화면으로 이동
- 회의록이 하나도 없으면: "아직 저장된 회의록이 없어요. 새 회의록을 만들어보세요." 안내 문구 표시

다 됐다고 볼 기준: 목록이 날짜 역순으로 보이고, 새 회의록 버튼이 동작함

## 5단계 — 프론트엔드: 새 회의록 작성 화면

화면: /meetings/new  
지금 모습: 없음 (신규 생성)  
바꿀 동작:

### 입력 영역

| 필드 | 타입 | 비고 |
|---|---|---|
| 회의 제목 | 텍스트 입력 | 필수 |
| 날짜 | 날짜 선택 | 필수, 기본값 오늘 |
| 참석자 | 텍스트 입력 | 쉼표로 구분, 선택 |

### 내용 입력 방법 선택 (탭으로 전환)

**[음성 파일 업로드] 탭**
- 파일 선택 버튼 (허용 확장자: mp3, m4a, wav, ogg, webm)
- 최대 파일 크기: 25MB
- 선택된 파일명 표시

**[텍스트 직접 입력] 탭**
- 여러 줄 텍스트 입력창 (높이 넉넉하게)
- 플레이스홀더: "클로바 노트 등에서 변환된 텍스트를 여기에 붙여넣어 주세요."

### AI 요약 버튼

- 버튼명: "AI 요약 시작"
- 색상: 골든 허니 (#F4A93C) 배경, 흰 글씨
- 클릭 시:
  1. 로딩 상태로 전환: "AI가 내용을 분석하고 있어요... 잠시만 기다려주세요" (스피너 표시)
  2. POST /api/meetings/transcribe 호출 (FormData로 audio 또는 text 전송)
  3. 성공 시: 요약 결과를 화면 하단에 펼쳐 보여줌 (마크다운 렌더링)
  4. 실패 시: 에러 메시지 표시

### 요약 결과 표시 영역 (AI 요약 후 노출)

- 배경: 연한 허니 베이지 (#FFF8EC)
- 섹션 구분: 핵심 내용 / 결정사항 / 할일 목록
- 수정 불가 (읽기 전용)

### 저장 버튼

- 요약 결과가 나타난 후 활성화
- 클릭 시 POST /api/meetings 호출 후 목록 화면으로 이동

다 됐다고 볼 기준: 파일 업로드 또는 텍스트 입력 → AI 요약 클릭 → 요약 결과 표시 → 저장 → 목록에 새 항목이 보임

## 6단계 — 프론트엔드: 회의록 상세 화면

화면: /meetings/:id  
지금 모습: 없음 (신규 생성)  
바꿀 동작:

- GET /api/meetings/:id 호출해서 데이터 렌더링
- 상단: 회의 제목 / 날짜 / 참석자
- 중간: AI 요약 결과 (마크다운 렌더링, 섹션별로 표시)
- 하단: "원본 전문 보기" 접기/펼치기 버튼 → raw_text 표시
- 우상단: "삭제" 버튼 → 확인 모달 → DELETE /api/meetings/:id → 목록으로 이동
- 뒤로가기 버튼: 목록으로 돌아가기

다 됐다고 볼 기준: 요약이 마크다운으로 보이고, 원본 접기/펼치기 동작하며, 삭제 후 목록에서 사라짐
