# 바꿈 허브 — 사용성 개선 업데이트 작업지시서 (2차)

**대상:** bakkum-class 통합 허브 (현재 이지현=원장/수학 적용됨, 데스크·초등영어·중고등영어 계정 존재)
**목적:** 실제 사용 동선 기준으로 가독성·편의·역할별 화면을 다듬는다.
**관계 문서:** 「바꿈 통합 허브 작업지시서」(구조·역할) 기준 위에 이번 개선 사항을 얹는다.

> ✅ 이미 반영됨(확인): 사이드바 "이지현님, 담당: 수학 · 원장" 표시 · "수학 수업관리" 카테고리명 · 학생 수정 모달(수업 스케줄 포함) · 강사 관리(화면 배정 체크박스).

> ⚠️ **절대 원칙 — 수학 불변:** 수학은 이미 잘 만들어졌다. 영어를 추가·통합하는 과정에서 **수학의 화면·데이터·동작이 바뀌거나 깨지면 안 된다.** 공통 요소를 손볼 때도 매 단계 수학 회귀 확인. **영어 = 수학을 복사한 '틀'에 내용만 영어로 채우는 방식**(UI 컨셉 동일, 내부 구성만 영어).

---

## 1. 여백/가독성 — "글이 선에 딱 붙는" 문제 (전역) ⭐
- **문제:** 카드·행·섹션에서 글 시작이 구분선/모서리에 바짝 붙어 답답하고 읽기 불편(첨부 이미지처럼).
- **개선:**
  - 카드 안쪽 여백 **최소 16~20px** 확보. 섹션 제목·본문이 테두리/구분선에 닿지 않게.
  - 표(테이블) 셀 좌측 패딩 **12~16px**, 행 높이 넉넉히.
  - 구분선과 다음 글 사이 **12px 이상** 간격.
  - 라벨↔값, 항목↔항목 간 8px 리듬 유지.
- *완료 기준:* 어느 화면이든 글이 선·모서리에 붙지 않고 숨 쉬는 여백이 있다.

## 2. 학생 명단 (이름 변경 · 공통 · 프로필형 수정) ⭐
- **이름 변경:** '학생 마스터/학생 관리' → **「학생 명단」** 으로 통일. **전 과목 공통**(한 곳에서 관리, 수학·영어 공유).
- **수정 = 세세하게:** 목록엔 핵심만, 나머지는 **수정**에서 깊게. 시간표·**등원 요일**·연락처·학교·상태·생년월일 등 모두 수정 가능.
- **프로필(이력서) 느낌 UI:** 단순 폼 나열이 아니라 **학생 프로필 카드**처럼.
  - 상단: 이니셜 아바타 + 이름 + 배지(학년·학교·상태·수강 과목 수학/영어).
  - 섹션 구분: ① 기본 정보 ② 연락처 ③ 수강·스케줄(과목별 요일·시간) ④ 이력/특이사항(누적).
  - 보기 좋게 카드·섹션으로 정리, 편집은 인라인 또는 '편집' 토글.
- *완료 기준:* '학생 명단'에서 학생을 누르면 이력서처럼 정돈된 프로필이 뜨고, 거기서 시간표·등원요일 포함 모든 정보를 수정·저장할 수 있다.

## 3. 영어 월말리포트 — 학생별 저장 · 미리보기 (초등영어 전용) ⭐
- **수학 월말리포트와 같은 흐름/UI**, 단 **리포트 내용은 영어에 맞춤**(8개 평가항목 Listening·Speaking·Reading·Spelling+Writing·Comprehension·Task Performance·Learning Attitude·Confidence, 등급 P~NI + Comments).
- **초등영어만** 대상(중고등 영어 리포트 없음).
- **담당 교사 선택:** 현재 **초등영어 담당 강사 목록에서 드롭다운 선택**(이름 직접 입력 대신) → 편의성 강화.
- **학생별 저장:** 전체 일괄 저장 + **학생 개별 저장** 모두 가능.
- **미리보기:** 수학처럼 저장 전에 **성적표 미리보기**로 확인.
- 데이터: 노션 「초등 리포트 DB」 / 성적표 시스템(`dndsu2810.github.io/bahkkum-k/english-report/dist`) 기반.
- *완료 기준:* 월·담당교사 선택 → 미리보기 확인 → 학생별/전체 이미지 저장.

## 4. 학원 일정 — 전원 열람·수정
- 학원 일정은 **모든 역할이 보고 수정**할 수 있게(공용). 누가 추가/수정해도 모두에게 반영.
- *완료 기준:* 강사·데스크·원장 누구나 일정 추가·수정 가능.

## 5. 영어(초등·중고등) 화면 — 수학과 동일한 틀, 내용만 영어 ⭐
영어 수업관리 카테고리에 **수학과 똑같은 메뉴 세트**를 둔다. **UI 컨셉·레이아웃·위치는 수학과 동일**, 안의 항목만 영어에 맞춤(노션 영어 구성 기준).

| 메뉴 | 수학(기준) | 영어 내용(맞춤) |
|---|---|---|
| **오늘** | 등원 학생 출결·숙제·진도 | 등원 학생 **일일기록**(학습 목표·숙제·숙제검사·코멘트·자료배부) |
| **시간표** | 수학 수업 요일·시간 | 영어 수업 요일·시간 |
| **출결 기록** | 출석/지각/결석 | 영어 출결(등원 선택 중심, 영어 특성 반영) |
| **숙제 기록** | 유형 칩 + 검사 | **자유 입력 숙제** + 숙제검사 |
| **진도 기록** | 단원 | **교재명+레벨**(자유 입력) |
| **테스트 기록** | 경시 등 점수 | **단어시험 점수**·평가 |
| **보강 관리** | 결석→보강 | 동일 흐름, 영어 수업 기준 |

- **UI 일관성:** 화면 구조·버튼 위치·카드 모양은 수학과 같게. 그래야 나중에 원장님이 코드를 가져가도 **틀은 그대로, 내용만 영어**로 유지·수정하면 됨.
- 초등영어·중고등영어 각각 동일 메뉴 세트(학년 구분만).
- *완료 기준:* 영어 강사 로그인 시 수학과 똑같은 자리·형태로 오늘·시간표·출결·숙제·진도·테스트·보강이 작동하고, 내용은 영어식이다. **그리고 수학 화면은 전혀 영향 없음.**

## 6. 원장 전용 대시보드 ⭐
> 목적: 원장이 실시간으로 다 보기 어려우니, **강사들 기록이 자동으로 모여 학생 관리가 되는** 화면.
- **이번 달 등록/재적 현황** + **지난달 대비 비교**(↑↓ 증감).
- **지각·결석 현황**(학생별/전체).
- **영수 합산 월별 리포트:** 한 학생의 수학·영어 기록·특이사항을 **월별로 한데 모아** 전반 관찰.
- **학생 개별 리포트:** 학생 클릭 → 그 학생의 영수 종합(출결·숙제·진도·테스트·특이사항).
- 데이터는 **강사들 일상 기록에서 자동 집계**(원장이 따로 입력 안 함).
- *완료 기준:* 원장 대시보드 한 화면에서 등록현황·증감·지각결석·학생별 영수 종합이 보인다.

## 7. 데스크쌤 — 오늘 화면
- 데스크에 **오늘 화면**: **현재 등원 학생 · 지각 학생**을 한눈에.
- 운영 보조용(조회 중심), 전체 시간표·학생 정보와 연결.
- *완료 기준:* 데스크 로그인 첫 화면에서 오늘 등원/지각 현황을 바로 본다.

## 8. 시간표 충돌 → 변경 요청·승인 시스템 ⭐
- 특정일/일정 기간만 **수업시간 임시 변경**이 필요할 때, 한 학생의 **영어↔수학 시간이 겹치면**:
  1. **"시간표가 겹칩니다" 팝업** 표시(어느 수업과 겹치는지).
  2. **"시간표 변경 요청하기"** — 보낼 대상을 두 가지로:
     - **담당 선생님에게 보내기**: 겹친 수업의 담당 교사에게 자동 발송.
     - **선생님 지정하기**: 강사 목록에서 **원하는 사람을 직접 골라** 발송.
  3. 선택한 선생님께 **앱 내 알림**(요청 내용·날짜·바뀔 시간 표시).
  4. 선생님이 **승인**하면 시간 교체 반영(거절/대안 제시도 가능).
- 임시 변경은 기간 종료 후 원복(원래 시간으로).
- *완료 기준:* 겹침 감지 → (담당/지정) 요청 → 해당 교사 승인 → 반영까지 동작.

## 9. 강사 업무 보드 — 카드 상세·필드·수정 ⭐
- **카드 클릭 → 상세 열림:** 카드를 누르면 **세부 지시 내용**(설명·메모)을 볼 수 있게.
- **입력 필드(카드 작성 시):**
  - 제목
  - **세부 지시 내용**(여러 줄 메모)
  - **담당자**(강사 지정)
  - **마감일**
  - **우선순위 = 급한 일 / 일반** 구분(급한 일은 눈에 띄게 표시: 빨강 라벨 등)
- **우선순위 정리:** 급한 일이 위로 오거나, 우선순위·마감일 기준 **정렬/필터**.
- **수정 가능(현재 안 됨 → 수정):** 카드의 모든 항목(제목·내용·담당자·마감일·우선순위)을 **나중에 수정**할 수 있게. (지금은 수정이 안 되는 상태로 보임 → 반드시 고칠 것)
- 칸반 3칸·드래그·완료 자동보관·전 강사 실시간 공유는 기존 사양 유지.
- *완료 기준:* 카드 클릭 시 상세가 보이고, 담당자·마감일·급함 여부를 넣어 우선순위를 정리할 수 있으며, 작성 후 수정도 된다.

## 10. 노션 ↔ 앱 — 단일 출처로 정리 ⭐
- **원칙:** 이제 **모든 일상 기록은 앱 한 곳**에 둔다. **노션에 중복 저장하지 않는다.**
- **끌어오기(import)는 1회만 OK:** 기존 노션 데이터를 앱으로 **한 번 가져오는 것**은 가능. 단, 이후 **양쪽에 같은 걸 계속 저장하는 중복은 없앤다.**
- **앱이 주인(매일 기록):** 학생명단 · 출결·숙제·진도·테스트 · 월말리포트 · 대시보드 · 할 일(업무 보드) · 생일 알림 · 특이사항 · 학원 일정.
- **노션이 주인(자료·문서):** 강사 자료실(커리큘럼·Q&A·가이드) · 신규 학생 메뉴얼 · 교재 구매 요청 · 마케팅 자료 → 앱 '바꿈 매뉴얼'에서 **읽기로 연결**.
- *완료 기준:* 같은 데이터가 노션·앱 양쪽에 중복 저장되지 않고, 일상 기록은 앱에서만 이뤄진다.

## 작업 순서 (로드맵)
1. **여백/가독성 전역 정리**(1번) — 가장 먼저, 체감 큼.
2. **학생 명단 + 프로필형 수정**(2번).
3. **영어(초등·중고등) 수학 동일 메뉴 세트**(5번) + **학원 일정 공용**(4번).
4. **영어 월말리포트 학생별·미리보기**(3번).
5. **강사 업무 보드 카드 상세·수정**(9번).
6. **원장 대시보드**(6번) + **데스크 오늘**(7번).
7. **시간표 충돌 요청·승인**(8번).
8. **노션 ↔ 앱 단일 출처 정리**(10번) — 중복 제거.
9. 각 단계 **수학 기존 기능 회귀 확인**.

## 확인 필요
- 시간표 변경 요청 시 알림 방식(앱 내 알림 vs 카카오워크) — 봇 보류 중이므로 우선 **앱 내 알림**으로.
- 원장 대시보드의 '특이사항 집계' 범위(이번 달/누적).
