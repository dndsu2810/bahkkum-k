# 바꿈영수학원 · 수업 관리 — 업데이트 지시서

> 작성: Claude · 검토: 지현T · 2026. 6. 16. (중고등 영어 신규 요청 반영)

이 문서는 바꿈영수학원 수업 관리 사이트를 직접 둘러보고, 실제 선생님·원장님들이 남긴 '오류·개선 요청'과 추가로 전달된 중고등 영어 관련 요청까지 확인한 뒤 정리한 것입니다. 무엇을 먼저 손보면 좋을지 순서대로 적었고, 작업하시는 분이 바로 이해하실 수 있도록 **어떤 화면 / 지금 모습 / 바꿀 동작 / 다 됐다고 볼 기준**을 함께 담았습니다.

## ⭐ 중고등 영어 전용 업데이트 (이번 신규 요청)

**왜 따로 묶었나:** 중고등 영어는 같은 학교 학생이어도 문제집에 들어가는 시기, 진도, 숙제 양이 학생마다 모두 다릅니다(예: 이유리·장진혁은 같은 판곡중이지만 들어간 과·숙제가 서로 다름). 그래서 수학처럼 **매번 숙제가 달라** 선생님들이 입력을 힘들어합니다. 아래 세 가지가 그 해결책입니다.

### A. 숙제 '학생별 개별 입력' — 순위 1

- **화면:** 오늘 (중고등 영어)
- **지금 모습:** 학습 목표 / 숙제 / 숙제 검사 / 코멘트 형태로 자유 입력은 되지만, 학생마다 매번 다른 숙제를 한 명씩 빠르게 넣고 이어 보기에는 부족함.
- **바꿀 동작:**
  - '오늘' 화면에서 **학생을 고르면 그 학생만의** 학습 목표 · 숙제 · 숙제 검사 · 코멘트를 자유롭게 개별 입력.
  - 공통 양식을 강제하지 않음(학생마다 항목 수·내용이 달라도 됨).
  - **지난 회차의 '숙제'가 이번 회차의 '숙제 검사'로 자연스럽게 이어져 보이게**(예: 지난번에 낸 '3과 그래머빌드업'이 이번 숙제 검사 칸에 미리 떠서, 미흡/완료만 체크).
- **다 됐다고 볼 기준:** 같은 시간대 학생이라도 서로 다른 숙제를 각각 입력·저장할 수 있고, 지난 숙제가 이번 '숙제 검사'로 이어져 보임.
- **참고:** 첨부한 이유리·장진혁 기록 형식(학습 목표 → 숙제 → 숙제 검사 → 코멘트)을 그대로 살리되, 학생별로 빠르게 넣을 수 있게만 다듬으면 됩니다.

### B. 자료배부 기준으로 숙제·진도 연결 — 순위 1

- **화면:** 자료배부 / 오늘 / 진도 (중고등 영어)
- **배경:** 프린트 자료를 ① 강사들과 공유 → ② 배정 → ③ 학생에게 배정 → ④ 거기서 숙제까지 연결되는 흐름입니다. 학생마다 다 다른 중고등 영어에서 **그나마 통일이 가능한 지점이 '프린트 자료'**입니다. 그래서 자료를 기준으로 숙제·진도를 배정하면 한결 수월합니다.
- **바꿀 동작:**
  - 자료배부 항목에 단계: ① 자료 등록(프린트물 이름) → ② 강사 공유·배정 → ③ 학생 배정.
  - **학생에게 배정한 자료가 그대로 '숙제'와 '진도'로 연결되게** 함. 자료를 고르면 그 학생 오늘 화면의 숙제·진도 칸에 자동으로 채워지거나 선택되도록.
  - 즉 **"이 프린트 자료 = 이 학생의 숙제·진도"** 로 한 번에 묶임.
- **다 됐다고 볼 기준:** 자료 하나를 학생에게 배정하면, 그 학생 오늘 화면의 숙제·진도에 그 자료가 자동으로 연결되어 따로 또 적지 않아도 됨.

### C. 내신기간 모드 ('내신모드') — 준비중 (계획만 먼저)

> **상태: 준비중.** 중고등 영어 카테고리에 '내신모드'를 추가하되, 지금은 **만들지 말고 계획만** 잡아 둡니다. 지현T가 원장님과 더 구체적으로 논의한 뒤 본 작업을 시작합니다.

- **위치:** 중고등 영어 카테고리에 '내신모드' 항목 신설(표시는 '준비중').
- **목적:** 내신(시험)기간에는 평소 진도 대신 **학교별 시험범위 중심 학습**으로 바뀌어야 함. 그 기간 동안 '오늘' 화면에 뜨는 항목을 평소와 다르게 보여주는 것.
- **핵심 동작(초안):**
  - **내신모드에 들어갈 학생을 선택(체크)** → 그 학생들만 '오늘' 화면 항목이 **내신모드용으로** 바뀜.
  - 학생마다 학교·시험기간이 다르므로, **학생별로 내신모드 시작일·종료일을 따로** 설정.
  - 종료일이 지나면 **평소 모드로 자동 복귀**.
- **내신모드 화면에 들어갈 항목(원장님 확정 전 초안):**
  - 학교 / 시험범위
  - 시험일(D-day 표시)
  - 시험범위별 진도 체크
  - 자료배부(학교별 프린트물) — 위 B와 연결
  - 오답·재시(NP) 관리
  - 시험 후 결과 기록
- **다음 단계(원장님과 확정할 것):**
  1. 내신모드에 꼭 들어갈 항목 최종 확정
  2. 시험범위를 어떻게 입력할지(학교별/학년별 틀)
  3. 평소 모드 ↔ 내신모드 전환 규칙(자동/수동, 알림 여부)
- **메모:** 위 셋이 정해지면 본 개발 착수. 그전까지는 화면에 '준비중'으로만 노출.

## 지금 잘 되어 있는 점 (그대로 유지)

새로 만드는 게 아니라 이미 좋은 부분이 많습니다. 아래는 유지하면 좋은 강점입니다.

- **'오늘' 한 화면에서 출결·태도·숙제 검사·내줄 숙제를 한 번에 입력** — 수업 직후 빠르게 기록하기 좋습니다.
- **출결 화면의 키보드 단축키**(1 출석 · 2 지각 · 3 결석 · ← → 다음) — 손이 빠른 선생님께 매우 편리합니다.
- **과목별(수학 · 영어 중고등 · 영어 초등)로 깔끔하게 나뉜 메뉴 구성.**
- **대시보드**의 요일별 분포 · 재적 현황 · 인센티브 정산 · '리포트 복사' 같은 운영자용 한눈 보기.
- **어두운 모드(다크모드)**가 색 깨짐 없이 일관되게 적용됨.
- **'오류·개선 요청' 화면 자체** — 사용자가 스크린샷과 함께 바로 의견을 남기고 처리 상태(접수·해결중·완료)를 볼 수 있는 점이 훌륭합니다.

## 순위 1 — 지금 바로 (공통)

### 1-1. '오늘' 화면에서 테스트 여러 개 입력 + 재시(NP) 표시

- **화면:** 오늘 / 테스트 기록
- **지금 모습:** '오늘' 화면에서 테스트를 한 개만 적을 수 있고, 단어·문장 시험에서 '통과 못 함'을 표시할 칸이 없음.
- **바꿀 동작:** '오늘' 화면에 테스트를 여러 개 추가할 수 있는 '＋' 버튼을 둠. 각 테스트마다 '통과 / 재시(NP)'를 고르는 버튼을 추가. 단어·문장 시험에서 미통과면 '재시(NP)'로 표시되도록.
- **다 됐다고 볼 기준:** 한 학생에게 테스트 2개 이상을 입력·저장할 수 있고, '재시(NP)' 선택이 기록과 리포트에도 그대로 나타남.
- **메모:** 정민아 원장님이 직접 요청(현재 '해결중').

### 1-2. 영어·수학 출결(포인트) 기록 방식 통일

- **화면:** 출결 기록 (영어 / 수학)
- **지금 모습:** 수학과 영어의 출결 기록 방식·항목이 서로 달라 헷갈림.
- **바꿀 동작:** 영어 출결 화면도 수학과 똑같은 구조로 통일. 출석/지각/결석 버튼, 수업 태도, 특이사항, 포인트 기록의 위치·색·이름을 동일하게 맞춤.
- **다 됐다고 볼 기준:** 수학과 영어 출결 화면을 나란히 놓았을 때 버튼·항목·색이 똑같아 보임.
- **메모:** 정민아 원장님 요청(현재 '해결중').

### 1-3. 휴대폰 화면 정리 (가장 큰 디자인 숙제)

- **화면:** 전체 (특히 '오늘' · 학생 명단)
- **지금 모습:** PC용 왼쪽 메뉴가 휴대폰에서는 위쪽 큰 자리를 로고가 차지하고, 메뉴는 좁은 칸에 몰려 들어감. 수업 중 휴대폰·태블릿으로 쓰기 번거로움.
- **바꿀 동작:** 휴대폰에서는 ① 위쪽 로고 영역을 작게 줄이고, ② 자주 쓰는 메뉴(오늘·출결·시간표·학생)를 화면 '아래 고정 탭' 또는 '≡(햄버거) 서랍'으로 빼고, ③ 명단 같은 표는 한 줄씩 카드 형태로 보이게.
- **다 됐다고 볼 기준:** 휴대폰에서 '오늘' 입력을 스크롤 한두 번으로 끝낼 수 있고, 메뉴 이동을 한 손으로 할 수 있음.
- **비유:** 지금은 큰 책상(PC)에 맞춘 서류함을 작은 가방(휴대폰)에 그대로 욱여넣은 모습입니다. 가방에는 가방에 맞는 정리함이 필요해요.

## 순위 2 — 곧

### 2-1. 학생 명단 '상태별 보기' + 휴대폰 표 정리

- **화면:** 학생 관리(명단)
- **바꿀 동작:** 재원 / 퇴원 / 휴원을 골라 볼 수 있는 버튼 추가(이전에 요청된 항목이니 잘 작동하는지 함께 확인). 좁은 화면에서 표가 깨지지 않게 카드형으로.
- **함께 점검:** 지금 명단 오른쪽 끝의 '수정' 글자가 화면 밖으로 살짝 잘려 보이는 현상(가로로 넘침)이 있으니 같이 고침.
- **다 됐다고 볼 기준:** 상태를 골라 명단이 바로 걸러지고, 휴대폰에서도 '수정' 버튼까지 안 잘리고 다 보임.

### 2-2. 자동 저장 안정성 재점검 + '저장됨' 표시

- **화면:** 오늘 · 출결 · 학생 화면 등 입력이 있는 곳 전체
- **배경:** 예전에 '자동 저장 오류'가 보고된 적이 있음(완료 처리됨). 입력 내용은 절대 날아가면 안 되는 부분이라 한 번 더 단단히.
- **바꿀 동작:** 입력 도중 새로고침하거나 인터넷이 잠깐 끊겨도 내용이 보존되도록 확인. 저장이 끝나면 '저장됨 ✓' 같은 작은 표시를 보여줌.
- **다 됐다고 볼 기준:** 입력 → 새로고침 → 내용 그대로 남아 있음. 저장 여부가 화면에 분명히 보임.

### 2-3. 알림 종에 '할 일' 모아 보여주기

- **화면:** 상단 알림(종) 아이콘
- **바꿀 동작:** 처리 안 된 개선 요청, 보강 대기 학생, 재시(NP) 대상, 마감 임박 업무를 종 안에 한데 모아 보여줌. 누르면 해당 화면으로 이동.
- **다 됐다고 볼 기준:** 종만 눌러도 '오늘 챙겨야 할 것'이 한눈에 보임.

### 2-4. '홈' 메뉴 정리

- **화면:** 왼쪽 메뉴의 '홈'
- **배경:** 지현T께서 직접 '홈이 왜 있는지 모르겠다, 없애도 될 듯'이라고 남기셨음.
- **바꿀 동작:** 홈을 없애거나, 눌렀을 때 곧바로 '오늘'로 가도록 연결. 메뉴를 한 칸 줄여 더 단순하게.
- **다 됐다고 볼 기준:** 들어오자마자 바로 쓰는 화면(오늘)으로 연결되어 헷갈림이 없음.

## 순위 3 — 여유 될 때 (다듬기)

### 3-1. 안내 문구·라벨을 쉬운 말로

- **바꿀 동작:** 업무 보드의 '노션 상태: 업무 배정'처럼 어려운 표현은 '담당자 정하기' 같은 쉬운 말로. 상단 '공지도 등록이 가능하지요~!' 배너는 한 번 닫으면 그날은 다시 안 뜨게.
- **이유:** 선생님들이 모두 컴퓨터를 잘 아는 건 아니므로, 화면 속 말은 학원 일상 용어로.

### 3-2. 색·여백·터치 영역·단축키 통일

- **바꿀 동작:** 어두운 모드에서 초록 '출석' 글자가 잘 보이는지 대비 확인. 작은 칩·버튼은 손가락으로 누르기 좋게 충분히 크게(권장 최소 44px). 과목이 달라도 같은 상태(출석/결석 등)는 같은 색으로.
- **추가:** 출결의 1·2·3·← → 단축키가 편하니, 진도·테스트 입력 화면에도 같은 단축키를 넓혀 적용.

## 디자인 기본 원칙 (작업 내내 기준 삼을 것)

- **색의 약속:** 파랑=기본 동작, 초록=출석·완료, 빨강=결석·주의, 노랑=강조·마감. 지금 체계가 좋으니 그대로 지킴.
- **휴대폰 먼저:** 선생님은 수업 중 휴대폰을 가장 많이 봄. 새 화면을 만들 땐 휴대폰에서 먼저 잘 보이는지 확인.
- **한 화면 한 일:** '오늘'처럼, 입력은 되도록 한 화면 안에서 끝나게.
- **어두운 모드 유지:** 지금처럼 밝은 모드와 똑같이 깔끔하게 보이도록 계속 점검.
