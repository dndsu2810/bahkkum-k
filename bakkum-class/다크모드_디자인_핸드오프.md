# 다크모드 디자인 핸드오프 — 바꿈 수업관리(bakkum-class)

**대상:** 이미 구현된 다크모드를 다듬는다. (토글 버튼 "어두운 모드로 전환" 이미 있음)
**목표:** 라이트모드 잔재(흰색 번짐·흰 입력칸)를 없애고, 카드가 또렷이 분리되며, 글씨·상태색이 눈 편한 다크 팔레트로 통일.
**원칙:** 순수 검정(#000) 금지. 살짝 푸른빛 도는 다크 슬레이트로 깊이감을 주고, **표면을 단계로 쌓아(layered surface)** 카드가 떠 보이게 한다.

---

## 🔎 현재 다크모드의 문제 (수정 대상)
| # | 문제 | 위치 |
|---|---|---|
| 1 | "오늘 한 줄 브리핑" 띠가 오른쪽에서 **흰색으로 그라데이션** | 오늘 화면 상단 히어로 |
| 2 | **카드가 배경에 묻힘** (카드/페이지 배경 명도 차 거의 없음) | 전 화면 카드 |
| 3 | **네이티브 날짜 입력칸이 흰색** 박스로 튐 | 숙제 마감일 등 date input |
| 4 | **시간·보조 글씨가 너무 어두워** 가독성 낮음 | 15:20 등 시간, 캡션 |
| 5 | **초록/파랑 강조색이 형광처럼 쨍함** | 출석 버튼, 선택 상태 |

---

## 🎨 다크모드 디자인 토큰 (이 값으로 통일)

### 배경 / 표면 (단계로 쌓기)
| 토큰 | 값 | 용도 |
|---|---|---|
| `dark-bg-sidebar` | `#0B0F18` | 사이드바(가장 어둡게, 깊이) |
| `dark-bg-base` | `#10151F` | 페이지/본문 바탕 |
| `dark-surface-card` | `#172033` | **카드 — 바탕보다 확실히 밝게(떠 보이게)** |
| `dark-surface-hover` | `#1E2A42` | 카드 hover, 칩 선택 전 |
| `dark-border` | `#26324A` | 카드·구분선 테두리(보이되 부드럽게) |
| `dark-border-strong` | `#33425F` | 입력칸·강조 테두리 |

> 핵심: **카드는 바탕(`#10151F`)보다 한 단계 밝은 `#172033`** 로. 지금처럼 같은 명도면 안 됨. 그림자는 다크에서 약하니 **표면 밝기 + 테두리**로 분리한다.

### 글자
| 토큰 | 값 | 용도 | 비고 |
|---|---|---|---|
| `dark-text-primary` | `#E8ECF4` | 본문/제목 | 순백(#FFF) 대신 살짝 톤다운 |
| `dark-text-secondary` | `#A7B0C0` | 보조·시간(15:20)·라벨 | 카드 위 명암비 ≥ 4.5:1 |
| `dark-text-caption` | `#8B97AD` | 가장 약한 캡션 | 명암비 ≥ 4.5:1 유지 |
> 지금 시간(15:20)·"70분·출석" 류가 너무 어둡다 → 최소 `dark-text-secondary`로 올린다.

### 포인트(파랑) — 다크에선 한 톤 밝게
| 토큰 | 값 | 용도 |
|---|---|---|
| `dark-accent` | `#6AA0FF` | 링크·강조 숫자·아이콘 |
| `dark-accent-solid` | `#3B82F6` | 채워진 버튼/선택 상태 배경(blue-600 대신 blue-500) |
> 라이트모드의 `#2563EB`를 그대로 쓰면 어두운 바탕에서 탁하다 → 텍스트형은 `#6AA0FF`, 채움형은 `#3B82F6`.

### 상태 색 — 채도 낮춘 다크 버전 (형광 방지)
| 의미 | 글자/아이콘 | 배경(반투명) |
|---|---|---|
| 출석·완료 | `#4ADE80` | `rgba(34,197,94,0.16)` |
| 지각·주의 | `#FBBF24` | `rgba(245,158,11,0.16)` |
| 결석·지연 | `#F87171` | `rgba(239,68,68,0.16)` |
> 라이트모드의 진한 초록/빨강(`#16A34A`/`#DC2626`)을 그대로 쓰면 쨍하다 → 다크에선 **400단계 + 반투명 배경**으로. 반투명이라 어떤 표면 위든 자연스럽게 얹힌다.

## 🧩 컴포넌트별 수정

### 1. "오늘 한 줄 브리핑" 히어로 띠 (가장 먼저)
- **현재:** 왼쪽 파랑 → 오른쪽 흰색 그라데이션 (다크에서 깨짐).
- **수정:** 흰색 끝을 없애고 **다크 그라데이션**으로.
  - 예: `linear-gradient(135deg, #1E2A4A 0%, #141A28 100%)` + 왼쪽에 은은한 파란 광 (선택).
  - 강조 숫자(5명·1건·5건)는 `dark-accent`(`#6AA0FF`). 본문은 `dark-text-primary`.
- *라이트모드에선 기존 그대로.* (테마별로 분기)

### 2. 카드 (오늘 테스트/학생 행/통계 카드 등)
- 배경 `dark-surface-card`(`#172033`), 테두리 `dark-border`(`#26324A`).
- hover 시 `dark-surface-hover`로 살짝 밝게(클릭 가능 신호).
- *완료:* 카드와 바탕이 한눈에 분리된다.

### 3. 네이티브 날짜 입력칸 (흰 박스 문제)
- 입력칸 자체: 배경 `dark-surface-card`, 글자 `dark-text-primary`, 테두리 `dark-border-strong`.
- **달력 아이콘/팝업까지 다크로:** 해당 input(또는 :root[data-theme=dark])에 **`color-scheme: dark;`** 적용. (이 한 줄이 네이티브 달력 흰 배경을 어둡게 바꿔줌)
- *완료:* 날짜칸·달력 팝업이 더는 하얗게 튀지 않는다.

### 4. 시간·보조 텍스트
- 15:20 같은 시간 라벨, "70분·출석", "6월·1-3단원" → `dark-text-secondary` 이상으로.
- *완료:* 어두운 바탕에서도 또렷.

### 5. 출결/태도 버튼·칩
- 출석 선택 = 상태색 출석(`#4ADE80` + 반투명 배경), 결석 = 결석색. 안 누른 버튼은 `dark-surface-hover` 배경 + `dark-text-secondary`.
- 숙제 칩(개념/연산…): 기본 `dark-surface-hover` + `dark-text-secondary` + `dark-border`, 선택 시 `dark-accent-solid`.
- "검사할 숙제"·"내줄 숙제" 라벨 칩: 반투명 틴트 배경 + 밝은 글자(위 상태색 규칙과 동일 톤).

## 🔁 테마 전환 (토글) 동작
| 항목 | 사양 |
|---|---|
| 토글 | 기존 해/달 버튼 유지. 아이콘은 현재 모드의 "반대"를 안내(현재 라이트면 달, 다크면 해). |
| 첫 진입 | 저장된 선택 없으면 **시스템 설정(prefers-color-scheme)** 따름. |
| 저장 | 사용자가 토글하면 선택 기억(다음 방문에도 유지). |
| 전환 애니메이션 | 배경·글자색 `200ms ease` 전환. **'동작 줄이기' 설정 시 전환 없이 즉시.** |

## ♿ 접근성
- 모든 글자: 해당 표면 위 **명암비 4.5:1 이상** (큰 제목·굵은 글씨는 3:1).
- 포커스 링: 다크에선 `#6AA0FF` 링 + 1~2px offset로 또렷하게.
- 상태는 색만으로 구분하지 말고 **글자/아이콘 라벨 병행**(출석/지각/결석 텍스트 유지).

## 🧪 점검 체크리스트
- [ ] 히어로 띠에 흰색 번짐 없음 (다크 그라데이션)
- [ ] 카드가 바탕과 명확히 분리(표면 한 단계 밝음 + 테두리)
- [ ] 날짜 입력칸·달력 팝업 다크 적용(`color-scheme: dark`)
- [ ] 시간·보조 글씨 명암비 4.5:1 이상
- [ ] 초록/파랑/빨강 형광 아님(400단계 + 반투명)
- [ ] 토글: 시스템 설정 따름 + 선택 기억 + 200ms 전환('동작 줄이기' 시 즉시)
- [ ] 라이트모드는 기존 그대로(회귀 없음)
