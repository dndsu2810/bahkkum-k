# 쏘이지 — 앱 화면 계획서

> 학생·선생님이 쓰는 쏘이지 앱 쪽 화면들입니다.
> 점수를 계산·저장하고, 영어 화면의 "수학 전광판" 칩과 상세 모달, 선생님 현황 화면을 담습니다.
> (학생이 키오스크에서 보는 화면은 "띵똥 — 수학 키오스크 화면 계획서"에 따로 있습니다.)

---

## 공통 전제 (반드시 지킬 것)

- 기존 숙제 기록과 노션 동기화는 **읽기만** 하고 절대 수정하지 않습니다. (명단: 노션→앱, 기록: 앱→노션 방향 유지)
- 기존 데이터는 손대지 않습니다. 수학 야구는 별도로 점수만 얹는 기능입니다.
- **시작일** 이후의 기록부터만 카운트합니다. (과거 미제출이 소급되어 아웃이 잔뜩 생기지 않게)

### 추천 기준값 (바꿀 수 있는 값)
| 항목 | 추천값 |
|---|---|
| 숙제 미제출 1회 | 스트라이크 1개 |
| 스트라이크 3개 | 아웃 1개 (S는 0으로) |
| 볼 4개 | 아웃 1개 삭제 (B는 0으로) |
| 아웃 3개 | 보충 수업 대상 |
| 카운트 초기화 | 매월 1일 |
| 볼 하루 적립 한도 | 최대 2개 |
| 볼 개수 | 추가숙제+1 / 자진등원+1 / 성적향상+2 / 모범태도+1 |

---

## 작업 A. 점수의 토대 (계산·저장, 화면 없음)

- **화면**: 없음 (점수를 세고 저장하는 토대 — 이 점수를 띵똥도 읽어 씁니다)
- **지금 모습**: 학생별로 스트라이크·볼·아웃을 세는 곳이 없습니다.
- **바꿀 동작**:
  - 학생마다 스트라이크(S)·볼(B)·아웃(O) 세 숫자를 저장합니다.
  - 숙제 미제출이 기록되면(시작일 이후) 자동으로 S를 1 올립니다.
  - S가 3이 되는 순간 → O를 1 올리고 S는 0으로.
  - 볼을 받으면 B를 1 올립니다. 단, 같은 날 이미 2개면 더 올리지 않습니다.
  - B가 4가 되는 순간 → 아래 순서로 처리하고 B는 0으로:
    1. 아웃이 있으면 → O를 1 내림
    2. 아웃 없고 스트라이크 있으면 → S를 1 내림
    3. 둘 다 없으면 → 꿀(쏘이지 포인트)로 전환
  - O가 3이 되면 → "보충 대상" 상태로 표시.
  - 모든 변화는 "언제 / 무엇 때문에 / 누가" 한 줄 기록으로 남깁니다.
- **다 됐다고 볼 기준**:
  - 숙제 미제출을 3번 넣으면 아웃이 1개 자동으로 생긴다.
  - 볼을 4개 주면 아웃이 1개 줄어든다.
  - 아웃 없는 학생에게 볼 4개를 주면 꿀로 바뀐다.
  - 매월 1일에 모든 숫자가 0으로 돌아간다.

---

## 작업 B. 영어 화면 — "수학 전광판" 칩 + 상세 모달

- **화면**: 초등영어(및 학생 화면) 위쪽
- **지금 모습**: 영어 내용만 있습니다.
- **바꿀 동작**:
  - 수학도 듣는 학생에게만 위쪽에 "수학 전광판" 칩(작은 알약 버튼, 야구공 아이콘)을 보여줍니다.
  - 수학을 안 듣는 학생에게는 칩이 아예 없습니다.
  - 칩을 누르면 **상세 전광판이 모달로 떠오릅니다**(미니 아님): 이름 + 큰 전광판 + 목표 한 줄 + 최근 기록.
  - 모달의 **X 버튼** 또는 **바깥 어두운 여백**을 누르면 닫힙니다.
  - 카드 **안쪽**을 눌렀을 때는 닫히지 않습니다. (실수로 닫히지 않게)
- **다 됐다고 볼 기준**:
  - 수학 듣는 학생에게만 칩이 보인다.
  - 칩을 누르면 상세 전광판이 뜬다.
  - X와 바깥 여백을 누르면 닫히고, 카드 안쪽을 누르면 안 닫힌다.
  - 볼의 점과 글씨가 초록이다.

### 모달 동작 상세 (변수의 변수)
- 열릴 때: 뒤 배경이 어두워지고 전광판 카드가 가운데에.
- 내용이 길면: 모달 안에서만 위아래로 넘기고 뒤 화면은 고정.
- 여러 번 눌러도 중복으로 쌓이지 않음.
- 폰 뒤로가기: 화면을 나가지 말고 모달만 닫힘.
- ESC 키로도 닫힘(있다면).
- 접근성: 열리면 초점이 모달로, 닫으면 칩으로 돌아옴. X에는 "닫기" 안내.

## 작업 C. 선생님용 — 반 현황 + 볼 주기

- **화면**: 선생님 메뉴에 "수학 야구" 새 화면
- **지금 모습**: 없습니다.
- **바꿀 동작**:
  - 반 학생 목록을 표로, 각자 S·B·O를 전광판 점(채움/빈칸)으로 표시.
  - 학생마다 상태 뱃지: 좋아요 / 주의 / 보충 위기.
  - 학생별 "볼 주기" 버튼 → 행동 선택: 추가숙제(+1)/자진등원(+1)/성적향상(+2)/모범태도(+1).
  - "스트라이크 취소" "아웃 면제" 버튼 + 한 줄 메모 (억울한 경우 대비).
- **다 됐다고 볼 기준**:
  - "볼 주기"를 누르면 그 학생 볼 점이 바로 늘어난다.
  - 취소/면제하면 숫자가 줄고 메모가 기록에 남는다.
  - 하루 2개를 넘겨 볼을 주려 하면 더 올라가지 않는다.

## 색과 부품

- 전광판 패널: 어두운 바탕 (#3A2A1A)
- LED 점: 볼 초록 #34C759(글씨 #5DD67F) / 스트라이크 주황 #FF7A3D / 아웃 빨강 #FF4438 / 꺼진 칸 #5A4630
- 수학 전광판 칩: 작은 알약 버튼, 꿀색 톤, 야구공 아이콘 + "수학 전광판"
- 전광판 모달: 어두운 배경 오버레이 + 가운데 상세 전광판 카드 + 우측 위 X
- 현황 표 한 줄: 이름 · S점 · B점 · O점 · 상태 뱃지 · 볼 주기 버튼
- 버튼: 기본(꿀색 채움, 흰 글씨) / 보조(흰 바탕, 테두리)
- 상태 뱃지: 좋아요(초록) / 주의(노랑) / 보충 위기(빨강)

## 빠질 수 있는 상황 (엣지 케이스)

- 점이 하나도 없을 때: 빈 점 + "아직 깨끗해요. 이대로 가요!"
- 이름이 길 때: 한 줄로 줄이고 … 처리
- 점수 불러오는 중: 회색 깜빡임
- 불러오기 실패: "잠시 후 다시 시도해 주세요" 안내
- 보충 대상인데 볼 4개가 막 채워질 때: 운영 계획서의 경계 규칙(보충 확정 전이면 차감 인정)을 따름

---

## 만들 순서 (한 번에 다 만들지 않기)

1. 작업 A(토대)로 점수가 제대로 오르내리는지 먼저 확인
2. 작업 B(칩 → 모달) 한 흐름을 프로토타입으로 만들어 지현님 확인
3. 확인되면 작업 C(선생님 현황)로 확장
4. (띵똥 화면은 작업 A가 끝나면 "띵똥 계획서"로 별도 진행)

> 새 디자인 토큰·컴포넌트를 만들지 말고 기존 공유 토큰·컴포넌트를 그대로 씁니다.
