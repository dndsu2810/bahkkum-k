# 바꿈 수업관리(bakkum-class) 통합 업그레이드 작업지시서 v3

**대상 앱:** bakkum-class (Cloudflare Workers + D1, 카카오워크 봇/웹훅 연동 경험 있음)
**목적:** 화면 개선(편의·미감·가독성)과 카카오워크 일일 브리핑 봇을 **하나의 지시서**로 묶는다.
**원칙:** 지금 토대(Pretendard 폰트 · 파란 포인트 · 16px 둥근 카드)는 좋으니 갈아엎지 말고 그 위에 얹는다.
**구성:** PART A 편의기능 · PART B 디자인 미감 · PART C 필수 개선 · PART D 카카오워크 봇 · PART E 강사 업무 보드

---

## 📐 디자인 토큰 (현재값 — 일관성 유지용)
- 폰트: `Pretendard Variable` (유지)
- 주 포인트색: `#2563EB`(blue-600), 진한 포인트 `#1D4ED8`(blue-700), 보조 `#7C3AED`(violet-600)
- 카드: 배경 `#FFFFFF` · 모서리 `16px` · 테두리 `#EAEEF4` · 그림자 `rgba(15,23,42,0.05) 0 1px 2px`
- 본문 글씨(정상): `#475569`(slate-600)

### ➕ 새로 정의할 "상태 색" 팔레트 (지금 출결 버튼이 무채색이라 상태가 한눈에 안 옴)

| 출석 · 완료 (성공) | `#16A34A` | `#DCFCE7` |

# 🟦 PART A. 편의 기능

**A-1. 로고 클릭 → 홈(오늘) 이동** — 손가락 커서 + hover 강조, 키보드로도 가능. *완료:* 어느 화면에서든 로고 누르면 오늘로.

**A-2. 상단 통계 카드 = 바로가기**
| 카드 | 이동 |
|---|---|
| 오늘 수업 | 오늘 등원 목록/시간표 |
| 출결 미체크 | 출결 기록(오늘) |
| 검사할 숙제 | 숙제 기록 |
| 보강 예정 | 보강 관리 |
| 오늘 테스트 | 테스트 기록 |
hover 시 "누를 수 있음" 신호(커서+미세하게 떠오름). *완료:* 카드 누르면 해당 화면 이동.

**A-3. 입력 후 토스트 + 되돌리기** — 처리하면 `최다연 · 출석 처리됨` 토스트 + 5초간 실행취소. 일괄 동작도 동일. *완료:* 처리 알림 + 한 번에 되돌리기.

**A-4. 날짜 이동 편하게** — 날짜 배지 옆 `‹ 어제 / 내일 ›` + `오늘로` 버튼. *완료:* 하루씩 이동·즉시 복귀.

**A-5. 학생 이름 클릭 → 학생 상세**(출결·숙제·진도 이력). *완료:* 이름 눌러 한 학생 흐름 확인.

**A-6. 빈 상태·로딩 상태 안내** — 데이터 없을 때 친절한 한 줄 + 다음 행동 제안, 로딩 중엔 회색 자리표시(스켈레톤). *완료:* 빈 화면·깜빡임 없음.

**A-7. (선택) 출결 키보드 단축키** — `1=출석·2=지각·3=결석`, `Enter=다음 학생`. *완료:* 키보드만으로 한 반 출결.

**A-8. (선택) 모바일 하단 탭바** — 좁은 화면에서 자주 쓰는 4~5개를 엄지 닿는 하단 고정. *완료:* 한 손 사용 가능.

# 🎨 PART B. 디자인 미감

**B-1. 상태를 색으로** — 위 팔레트로 출석=초록/지각=앰버/결석=빨강, 선택된 것만 채움. *완료:* 훑기만 해도 상태 구분.

**B-2. 글자 위계 정리** — 화면제목 22~24 Bold / 섹션제목 16~18 SemiBold / 본문 14 / 캡션 13(`#64748B`). 대시보드 큰 숫자는 더 또렷하게, 단위는 작게. *완료:* 크기·굵기로 위계 분명.

**B-3. 간격 리듬 통일(8px)** — 4·8·12·16·24·32 단위. *완료:* 간격 들쭉날쭉 없음.

**B-4. 미세 인터랙션** — 150~200ms 부드럽게(카드 hover 살짝 떠오름, 버튼 누름 피드백, 탭 전환). **'동작 줄이기' 설정 시 애니메이션 끔.** *완료:* 살아있되 산만하지 않음.

**B-5. 배지·태그 정리** — 학년 배지·행사 태그 색/모양/크기 한 세트로. *완료:* 한 디자인 언어로 보임.

**B-6. ✨ 시그니처 — "오늘 한 줄 브리핑"** — 오늘 화면 맨 위 한 줄 요약(`오늘은 4명 등원 · 출결 4명 대기 · 숙제 4건 검사 · 테스트 5건`), 숫자 누르면 해당 위치로(A-2 연결). ez쌤 톤으로. *완료:* 한 줄로 그날 파악.

**B-7. (선택) 다크 모드** — 헤더/설정에 토글. *완료:* 야간 눈부심 없음.

**B-8. 마이크로카피 통일** — 동작은 일어날 일 그대로("저장"→토스트 "저장됨"), 빈/에러는 사과 대신 안내. *완료:* 같은 동작 다른 말 안 씀.

# 🔴 PART C. 필수 개선 (가독성·접근성)

**C-1. 연한 회색 글씨 진하게 (최우선)**
- 보조 설명글 `#8A97A8`(명암비 2.97:1) → 최소 `#64748B`(약 4.75:1), 중요 안내는 `#475569`.
- breadcrumb `#B8C1CF`(1.82:1) → 최소 `#64748B`, 현재 위치 강조 `#334155`.
- *완료:* 모든 텍스트 명암비 4.5:1 이상.

**C-2. 시간표 겹침 시 이름 잘림** — 한 칸 너비 약 88px 미만이면 요약 칩(`정시우 외 2명`)으로 묶고 클릭 시 펼침. *완료:* 3개 겹쳐도 식별 가능.

**C-3. 버튼 터치 44px** — 숙제 칩·완성도·출결 버튼 터치영역 최소 44×44px(padding 확장 가능), 칩 간격 확보. *완료:* 태블릿 오터치 없음.

**C-4. 제목 구조 & 좁은 화면 로고** — 화면별 `<h1>` 1개 + `<h2>/<h3>` 순서, 본문 `<main>`. 좁은 화면에서 로고는 가로 스크롤 탭과 분리해 **고정(sticky)**. *완료:* h1 존재, 로고 안 사라짐.

**C-5. 시간표 빈 시간 정리** — 진입 시 첫 수업 근처로 자동 스크롤 또는 빈 앞 시간 접기. *완료:* 열면 실제 수업 바로 보임.

# 💬 PART D. 카카오워크 일일 브리핑 봇

봇은 데이터를 새로 만들지 않는다. **앱의 오늘자 데이터를 읽어 요약 → 카카오워크로 전송**만 한다.

## ✅ 확정 설정
| 항목 | 값 |
|---|---|
| 받는 곳 | **원장(ez쌤) 1:1** |
| 낮 브리핑 | **매일 13:00 (KST)** |
| 밤 요약 | **매일 21:00 (KST)** |
| 수업 없는 날 | **발송 안 함** |

## ⏰ 스케줄 메모
- Cron은 UTC 기준. 13:00 KST → `0 4 * * *`, 21:00 KST → `0 12 * * *`. (한국 서머타임 없음. "오늘 날짜"는 코드에서 KST로 계산.)

## 🚦 발송 조건
- 오늘 수업(정규+보강) ≥ 1건 일 때만 발송. 휴원일이면 발송 안 함. (낮·밤 동일)

---

## 📩 메시지 ① 낮 13:00 — "오늘 브리핑" (시작 전 체크리스트)

### 구성
0. **🎂 오늘 생일** — 오늘 생일인 재원생이 있으면 **맨 위에** 표시. (없으면 줄 자체를 생략)
   - 형식: `🎂 오늘 생일 · 김하린 — 등원할 때 축하해 주세요!`
   - 여러 명이면 이름 나열.
1. **오늘 등원 ({N}명)** — 시간순. `15:00 최다연 · 16:30 민서준, 정시우, 김하린`
2. **숙제 검사 ({N}건)** — `최다연 「개념책 풀어오기」 외 N건`
3. **보강·결석 예정** — `오늘 보강 1건 — 정시우 16:30` / 없으면 `예정 없음`
4. **특이사항** — `오늘 KTC수학경시대회 테스트 5명 · 신규 윤하영 첫 주` / 없으면 `없음`

### 텍스트 템플릿
```
[오늘 브리핑] {M}월 {D}일 ({요일})
오늘 하루, 미리 챙겨요

🎂 오늘 생일 · {이름} — 등원할 때 축하해 주세요!   ← 생일 학생 있을 때만
· 오늘 등원 ({등원수}명)
  {시간/이름 목록}
· 숙제 검사 ({검사건수}건)
  {학생 「숙제명」 … 외 N건}
· 보강·결석 예정
  {내용 또는 "예정 없음"}
· 특이사항
  {내용 또는 "없음"}

▸ 출결 바로 입력하기: {앱 링크}
```

> **생일 기능 준비물:** 학생별 **생일 정보**가 있어야 한다. 현재 학생 정보에 생일 칸이 없으면 **먼저 추가**(노션/D1에 '생일' 필드). 연도 없이 월·일만 있어도 됨. (보너스: 앱 '오늘 화면'에도 생일 학생에게 🎂 배지를 달면 더 좋음.)

## 📩 메시지 ② 밤 21:00 — "오늘 수업 요약" (마감 + 후속조치)

### 구성
1. **출결 결과** — `출석 3 · 지각 0 · 결석 1`. **미체크 남으면** 끝에 `· 미체크 N ⚠ 확인 필요`.
2. **보강 잡을 학생** — 오늘 결석 → 보강 미정 학생. `김하린 — 오늘 결석, 보강 미정` / 없으면 `없음`. (결석 학생 자동 등록)
3. **오늘 내준 숙제** — `최다연(6/15) · 민서준(6/16) 외 N명` / 없으면 `없음`
4. **특이사항** — `출결 미체크 1명 남음 · 정시우 진도 2단원 완료` / 없으면 `없음`

### 텍스트 템플릿
```
[오늘 수업 요약] {M}월 {D}일 ({요일})
하루 마감, 빠진 것 없나요

· 출결 결과
  출석 {N} · 지각 {N} · 결석 {N}{미체크 있으면 " · 미체크 N ⚠ 확인 필요"}
· 보강 잡을 학생
  {결석자/보강미정 목록 또는 "없음"}
· 오늘 내준 숙제
  {학생(마감일) … 외 N명 또는 "없음"}
· 특이사항
  {내용 또는 "없음"}

▸ 보강/출결 정리하기: {앱 링크}
```

## 🎨 봇 표시 규칙
- 빈 섹션도 `없음`/`예정 없음`으로 채움. (단, 🎂 생일 줄은 있을 때만)
- 결석·보강·미체크처럼 챙길 항목은 `⚠`·라벨·맨 위 배치로 강조(카카오워크 1:1은 색 강조 제한적).
- blocks 포맷 가능하면 header+divider+본문+**앱 링크 버튼** 구조로.
- 문구는 담백하게, 사과체·과장 금지.

## 🔧 봇 준비물 (설정값)
- 카카오워크 **봇 API 키/토큰**
- **원장(ez쌤) 식별자**(카카오워크 이메일/user id) — 1:1 발송용
- 앱 바로가기 **링크 URL**
- **학생 생일 데이터**(없으면 필드 추가 — 위 참고)
- (기존 키오스크·웹훅 봇 재활용 가능)

## 🧪 봇 테스트 방법
- 수동 실행 임시 주소(`/__send-noon`, `/__send-night`)를 만들어 즉시 발송 테스트. 문구·빈값·시간·생일 표시 확인 후 보호/제거.

# 🗂️ PART E. 강사 업무 보드 (칸반)

원장이 해야 할 업무를 카드로 관리하는 보드. 지라처럼 카드를 칸 사이로 끌어 옮긴다.

## ✅ 확정 설정
| 항목 | 값 |
|---|---|
| 위치 | 사이드바 **'오늘' 바로 다음** 메뉴 (이름: 강사 업무) |
| 칸 구성 | **3칸 — 할 일 / 진행 중 / 완료** |
| 사용 | **원장 1인 전용** (담당자/공유 없음) |
| 결석→보강 카드 | **자동 생성함** |

## 구성
- 카드 1장 = 업무 1개. **드래그로 칸 이동 → 상태 변경.** 완료 칸 카드는 취소선+체크, 살짝 흐리게.
- 카드 필드:
  - 제목 (필수)
  - 분류 태그 (선택): 보강 · 학부모 · 교재 · 경시 · 마케팅 · 행정 · 수업준비 등 — 색으로 구분
  - 마감일 (선택): `오늘`, `6/20` 식 표시. 지난 마감은 눈에 띄게.
  - 연결 학생 (선택): 카드를 특정 학생과 연결 (예: 보강 카드)
- 동작: **빠른 추가**(상단 한 줄 입력 → Enter) · 드래그 이동 · 수정 · 완료 · 삭제.
- (선택) 정렬·필터: 마감일순 / `오늘·이번주` / 태그별.

## 🔁 결석 → 보강 카드 자동 생성 (확정)
- **결석으로 처리되는 즉시** '할 일' 칸에 보강 카드를 자동 생성. (밤 21시 요약의 "보강 잡을 학생"과 같은 목록을 가리킨다)
  - 제목 `{학생} 보강 일정 잡기` · 태그 `보강` · 연결 학생 = 해당 학생 · 메모 "결석 후속".
- **중복 방지:** 같은 학생·같은 결석 건은 한 번만. 이미 있으면 안 만든다.
- **정정 시 회수:** 결석을 출석/지각으로 바꾸면, 자동 생성된 그 보강 카드는 자동으로 거둬들인다(삭제).
- 보강이 실제로 잡히거나 카드를 완료 처리하면 더는 띄우지 않는다.

## 📥 완료 카드 누적 방지 (보드가 밑으로 끝없이 길어지지 않게)
- **모든 칸은 최대 높이를 정하고, 카드가 많으면 칸 안에서만 스크롤.** 보드 전체가 세로로 늘어나 화면을 잡아먹지 않게 한다.
- **완료 칸 자동 보관:** 완료한 지 **7일 지난 카드는 '보관함'으로 자동 이동**해 보드에서 사라진다. 보관함에서 언제든 다시 볼 수 있다.
- **(선택) 완료 칸 기본 축소:** 완료 칸은 기본적으로 최근 완료만 보여주고, "전체 완료 보기"로 펼친다.
- **(선택) 수동 정리:** "완료 카드 정리" 버튼으로 완료 카드를 한 번에 보관.
- *완료 기준:* 완료 카드가 수십 개 쌓여도 보드가 한 화면 높이를 크게 벗어나지 않고, 지난 완료는 보관함에서 확인된다.

## 🗄️ 보관함 조회 (지난 완료 기록 다시 보기)
- **완료 칸 맨 아래 "보관함 보기" 링크** → 누르면 보관된 완료 카드 전체가 목록으로 열린다 (완료 날짜순, 태그별 정리).
- 보관함에서 **검색·필터** 가능: 태그별(예: 보강만), 기간별(예: 6월), 연결 학생별.
- 보관된 카드를 **다시 보드로 꺼내오기(되살리기)** 가능 — 잘못 보관했거나 다시 진행할 때.
- 보관 카드는 삭제되지 않고 계속 남는다(추적용). 비우고 싶을 때만 수동 삭제.
- *완료 기준:* "보관함 보기"로 지난 완료 기록을 날짜·태그·학생별로 찾아볼 수 있고, 필요하면 보드로 되살릴 수 있다.

## 완료 기준
- '오늘' 다음에 '강사 업무' 메뉴가 있고 보드가 열린다.
- 카드 추가 · 드래그 이동 · 수정 · 완료 · 삭제가 된다.
- 완료 카드는 취소선+체크로 구분된다.
- 결석생이 '보강' 카드로 자동 생성되며 중복되지 않는다.
- 완료 카드가 많이 쌓여도 보드가 밑으로 무한정 길어지지 않는다(칸 내부 스크롤 + 7일 후 자동 보관).
