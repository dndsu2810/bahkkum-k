# 바꿈 통합 허브 · 디자인 점검 & 보완 지시서 (Claude Code용)

> ## ⚠️ 먼저 읽으세요 — 이건 "새로 만드는" 지시서가 아닙니다
> 이 앱(`bakkum-class`)은 **이미 v2·v3 업그레이드가 적용된 거의 완성형**입니다.
> 디자인 토큰·토스트·스켈레톤·날짜이동·오늘 브리핑·출결 단축키 등 **대부분이 이미 구현돼 있습니다**(아래 §1 목록 + 코드 주석의 A-2/A-4/B-6/C-1 태그가 증거).
>
> **그래서 작업 방식은 딱 하나:**
> 1. **먼저 코드를 읽는다** (`src/styles.css`, `src/components/*`, `src/lib/*`).
> 2. 아래 §1 "이미 있음"은 **절대 새로 만들지 말 것** — 발견되면 그대로 둔다.
> 3. §2 "점검 항목"을 **하나씩 확인만** 하고, **실제로 빠졌거나 어긋난 것만** 고친다.
> 4. 손대기 전에 **"이건 이미 X에 있습니다 / 없어서 추가합니다"를 먼저 보고**하고 진행한다.
> 5. **기능 추가·재작성 금지.** 시각/일관성 보정만. 의심되면 멈추고 물어본다.

## §1. 이미 구현됨 — 건드리지 말 것 (확인용)

`src/styles.css`(약 178KB)와 컴포넌트에서 확인된 것들. 새로 만들지 말 것.

## §2. 점검 항목 — "확인 후, 빠진 것만"

각 항목은 **명령이 아니라 질문**입니다. 코드에서 확인 → 이미 있으면 PASS, 없거나 어긋나면 그것만 보정.

### 2-1. 일관성 점검 (가장 가치 큼, 안전함)
- [ ] **하드코딩 색 스캔.** `src/` 전체에서 `#[0-9a-fA-F]{3,6}` 검색 → `styles.css` 토큰 정의부와 `index.html` 깜빡임 방지용을 **제외**하고, 컴포넌트(tsx)에 직접 박힌 색이 있으면 가장 가까운 `var(--*)`로 치환. **단, 시각 결과가 바뀌면 안 됨**(같은 색의 토큰으로만).
- [ ] **이모지 잔여 스캔.** UI 텍스트/버튼/라벨에 이모지가 있으면 `icons.tsx`의 아이콘 또는 배지로 교체. (주석·문서의 이모지는 그대로 둠.)
- [ ] **배지/버튼 클래스 통일.** 같은 의미인데 다른 클래스를 쓰는 곳이 있으면 §1의 표준(`.badge.b-*`, `.btn.primary/ghost/danger`)으로 모음.
- [ ] **허브 신규 화면 vs 수학 화면 톤 비교.** 허브 화면(`.content.is-hub`)이 수학 화면과 카드 패딩·제목 크기·간격이 어긋나면 토큰 값으로 맞춤.

### 2-2. 라이트/다크 동등성
- [ ] 다크 테마에서 **대비 떨어지는 글자/테두리**가 있는지 훑기. 있으면 해당 `[data-theme="dark"]` 규칙만 보정(라이트는 건드리지 않음).
- [ ] 다크에서 하드코딩된 밝은 배경(`#fff`, `#f...`)이 카드 안에 남아 튀는 곳 확인.

### 2-3. 반응형 (좁은 화면)
- [ ] 880px 이하에서 사이드바/상단바/시간표/오늘 카드가 깨지는지 확인. CSS에 모바일 분기(`@media(max-width:...)`)가 이미 많으므로 **새 레이아웃 만들지 말고** 빠진 분기만 보완.
- [ ] 터치 타깃 44px가 안 지켜진 작은 칩이 있으면 padding으로만 확보.

### 2-4. 확인 필요 (구현 여부 불확실 — 있으면 PASS)
> 아래는 v3 지시서 항목 중 **코드에서 확실히 못 본 것**들. **먼저 있는지 확인**하고, 정말 없을 때만 별도 합의 후 진행. 임의 구현 금지.
- [ ] **업무보드 칸반**(할일/진행/완료 드래그) — `hubApi.ts`/관련 컴포넌트에 있는지?
- [ ] **결석 → 보강 카드 자동 생성 + 완료 7일 후 보관** — `MakeupList`/`hubApi`에 로직 있는지?
- [ ] **카카오워크 일일 브리핑 봇**(낮 13:00 / 밤 21:00, 원장 1:1) — 워커 cron(`wrangler.toml`/워커 엔트리)에 있는지?
- [ ] **학생 '생일' 필드**(봇 생일 안내 선행) — `migrations/012_student_meta.sql`에 포함됐는지 확인.

## §3. 절대 규칙 (가드레일)

1. **읽기 → 보고 → 수정** 순서 엄수. 큰 파일(`styles.css`, `modals.tsx`)은 통째 재작성 금지, 부분 수정만.
2. **시각 결과를 바꾸는 변경 금지** (토큰 치환은 "같은 색"일 때만). 색감·간격을 "개선"하지 말 것 — 요청은 *일관성*이지 *리디자인*이 아님.
3. **데이터 계층 불변**: `/api/health` 감지 → D1/localStorage 자동 전환, 노션 양방향, 모각공 `students` 공유 규칙(삭제·전체 덮어쓰기 금지) 그대로.
4. **수학 앱 영역(`.is-math` 등) 영향 주지 말 것.** 허브 전용 규칙은 `.content.is-hub` 스코프 유지.
5. 각 변경은 **작은 커밋 + 한 줄 설명**. 기존 기능(출결·숙제·시간표·노션) **회귀 테스트** 후 다음으로.
6. 조금이라도 "새 기능/재구조화"로 번지면 **멈추고 확인 요청**.
