# 바꿈 수업관리 — 통일 & 마감 업데이트 지시서

> 대상: `bakkum-class.dndsu2810.workers.dev` (GitHub: `dndsu2810/bahkkum`)
> 성격: **마지막 대규모 다듬기**. 새 기능 추가가 아니라, 흩어진 디자인을 하나로 묶고, 느린 느낌을 없애고, 보기 편하게 만드는 작업.

---

## 0. 이 업데이트 한 줄 목표

> **"같은 기능은 같은 모습으로, 모든 화면이 한 사람 손에서 나온 것처럼."**

지금 가장 아쉬운 점은 **통일감**이다. 기능이 나빠서가 아니라, 같은 일을 하는 화면인데 버튼·카드·간격·글자가 화면마다 제각각이라 "정리 안 된 느낌"이 든다. 이번 작업의 80%는 *새로 만드는 게 아니라, 이미 있는 것들을 하나의 규칙으로 줄 세우는 것*이다.

---

## 1. 대원칙 (모든 작업에서 항상 지킬 것)

이 5가지는 어느 화면을 손대든 매번 적용한다.

1. **같은 기능 = 같은 부품.** '저장', '추가', '날짜 선택', '학생 선택' 같은 공통 동작은 화면이 달라도 똑같이 생기고 똑같이 동작해야 한다.
2. **값은 토큰으로만.** 색·글자크기·간격·모서리·그림자는 정해진 토큰 표에서만 가져온다. 화면 안에 직접 박아넣은(하드코딩) 색·숫자는 전부 토큰으로 교체한다.
3. **접근성 최소 기준(WCAG AA) 충족.**
   - 글자 대비 4.5:1 이상 (큰 글자는 3:1 이상)
   - 누르는 영역 최소 44×44 (손가락으로 정확히 눌리게)
   - 모든 입력칸에 라벨, 모든 버튼에 명확한 이름
4. **문구 일관성.** 같은 것은 어디서나 같은 단어로 부른다. (예: '숙제' vs '과제' 섞지 않기 — 하나로 통일)
5. **데이터 연동은 절대 건드리지 않는다.** (아래 6번 주의사항 참고)

> **비유:** 레고를 떠올리면 된다. 집을 새로 지을 때마다 블록을 새로 깎지 말고, 정해진 블록 세트에서 가져다 쓴다. 같은 모양 블록은 색·크기가 항상 같다.

## 2. [Phase 1] 통일감 — 디자인 정비 (가장 먼저, 가장 중요)

### 2-1. 먼저 전체 화면 목록부터 만든다 (진단)

코드를 손대기 전에, **앱 안의 모든 화면과 그 안의 부품들을 쭉 훑어 목록**으로 정리한다. 어디가 어떻게 다른지 눈에 보여야 통일할 수 있다.

산출물 예시:

```
화면 목록 / 부품 통일 점검표
| 화면 | 쓰인 카드 종류 | 쓰인 버튼 종류 | 하드코딩 값 | 비고 |
| 오늘 | A형 카드 | 파랑 버튼 | 색 3곳 | - |
| 초등수학 보강관리 | B형 카드 | 초록 버튼 | 간격 5곳 | 보강관리인데 중고등과 다름 |
| 중고등 보강관리 | C형 카드 | 파랑 버튼 | - | 위와 통일 필요 |
...
```

### 2-2. 토큰 표 정리 (디자인의 '기본 재료')

이미 토큰 시스템이 도입돼 있다. 그걸 기준으로:
- 빠진 토큰을 채우고,
- 화면에 직접 박힌 값들을 전부 토큰으로 교체한다.

정리할 토큰 묶음:

| 묶음 | 내용 |
|------|------|
| 색 | 브랜드색 / 강조색 / 성공·경고·위험 / 글자색(진함·보통·흐림) / 배경·테두리 |
| 글자 | 크기 단계(제목·소제목·본문·작은글씨) / 굵기 / 줄간격 |
| 간격 | 4·8·12·16·24·32 같은 정해진 단계만 사용 |
| 모서리 | 카드·버튼·입력칸 둥근 정도 통일 |
| 그림자 | 떠 있는 정도(평평·살짝·많이) 3단계 |
| 움직임 | 화면 전환·버튼 반응 속도(예: 0.2초)와 부드러움 통일 |

### 2-3. 공통 부품을 하나로 통일

아래 부품들을 **각각 하나의 정의로** 통일하고, 모든 화면이 그걸 가져다 쓰게 한다.

- 버튼 (주요/보조/위험 — 3종류, 크기 3단계)
- 카드 (정보 카드 한 가지 모양으로)
- 탭 / 메뉴
- 리스트 행 (학생 목록, 보강 목록 등)
- 입력칸 / 날짜 선택 / 드롭다운
- 모달 (확인창)
- 토스트 (저장됨 같은 짧은 알림)
- **빈 상태** (목록이 비었을 때 보여줄 안내)
- **로딩 상태** (3-5번 Phase에서 함께)

### 2-4. ⭐ 보강관리 3종 통일 (지현님이 콕 집은 부분)

**초등수학 보강관리 / 중고등 보강관리 / 초등 보강관리** — 셋의 *안에 담기는 내용·구성이 다른 건 그대로 둔다.* (그건 원래 다른 게 맞음)

**하지만** 셋이 공유하는 같은 동작과 틀은 똑같이 맞춘다:
- '보강 추가' 버튼의 모양·위치·문구
- 날짜 선택 방식, 학생 선택 방식
- 보강 항목 카드/행의 생김새
- 저장·취소·삭제 버튼
- 화면 전체 레이아웃 골격 (제목 위치, 목록 영역, 여백)

> 정리: **"내용은 달라도 좋다. 입는 옷은 같아야 한다."**

## 3. [Phase 2] 홈 화면 결정 `[지현님 최종확인]`

### 진단 먼저
Claude Code는 먼저 **현재 홈이 무엇을 보여주는지** 확인한다. 판단 기준은 단 하나:

> "이 홈에, 지금 이 순간 학원 운영에 바로 쓸모 있는 정보가 있는가?"

### 추천 방향 (기본값으로 적용)
- '오늘' 화면이 이미 숙제 확인/내주기를 다루고 있으므로, 홈은 *메뉴를 한 번 더 거쳐가는 문* 역할에 그칠 가능성이 높다.
- **그렇다면: 홈을 없애고, 앱을 켜면 곧장 '오늘' 화면으로 진입.** 단, '오늘' 화면을 단순 숙제 화면이 아니라 **하루를 한눈에 보는 대시보드**로 키운다:
  - 오늘 수업 일정
  - 체크해야 할 숙제 / 아직 안 한 출결
  - 빠른 진입 버튼 (홈에 있던 메뉴들을 여기로 흡수)
- 만약 진단 결과 홈에 *진짜 유용한 정보*가 있었다면 → 없애지 말고 '오늘'과 **합쳐서** 하나의 강력한 첫 화면으로 만든다.

> 어느 쪽이든 결론은 동일: **첫 화면 = 오늘 할 일이 바로 보이는 곳. 군더더기 진입 단계 제거.**

⚠️ 이 항목은 **지현님 확인 후 진행**한다. (홈을 살리고 싶으면 그 의사를 우선한다)

## 4. [Phase 3] 성능 — 튕김·버벅임·한박자 늦음 없애기

지현님이 느낀 "화면이 튕기고, 버벅이고, 한박자 늦게 뜬다"는 거의 다 **데이터를 화면 들어갈 때마다 새로 불러오느라** 생기는 현상이다. 해결 방향은 다섯 가지.

| 증상 (지현님 표현) | 원인 | 해결 (Claude Code가 구현) |
|---|---|---|
| 화면이 한박자 늦게 뜸 | 들어갈 때마다 데이터를 처음부터 불러옴 | **한 번 불러온 건 기억(캐싱)** 후 즉시 보여주고, 뒤에서 조용히 갱신 |
| 빈 화면이었다가 갑자기 튀어나옴 | 데이터 오기 전 화면이 비어 있음 | **로딩 자리(스켈레톤)**를 먼저 깔아 빈→가득 점프 제거 |
| 화면이 밀리고 튕김 | 데이터가 나중에 들어오며 자리가 밀림 | **자리를 미리 고정**(높이·영역 예약)해 밀림 방지 |
| 버튼 눌러도 반응이 굼뜸 | 서버 응답을 기다린 뒤에야 화면 반영 | **누르면 즉시 반영**(낙관적 업데이트), 실패 시 되돌림 |
| 화면 전환이 딱딱·깜빡임 | 전환 효과 없음 | **부드러운 전환**(짧은 페이드 0.2초)으로 통일 |

> **비유들**
> - 캐싱 = 냉장고에 미리 채워두기. 손님 올 때마다 장 보러 안 나감.
> - 스켈레톤 = 음식 나오기 전 빈 접시·수저 먼저 세팅. 갑자기 안 튀어나옴.
> - 낙관적 업데이트 = 엘리베이터 버튼. 누르면 *바로* 불이 들어오고, 그다음 움직임.

추가로 점검:
- 같은 데이터를 여러 번 불러오는 곳이 있으면 한 번으로 줄이기
- 불필요하게 화면 전체를 다시 그리는 부분 줄이기 (바뀐 부분만 갱신)

## 5. [Phase 4] 수학 — 연간 수업 계획 개선 (지현님 요청)

수학 쪽 디자인은 마음에 든다고 하셨으니, **연간 수업 계획만** 집중 개선한다.

요구사항:
1. **연도별 보기 추가** — 연도를 골라(또는 ◀ ▶로 넘겨) 그 해의 계획을 볼 수 있게.
2. **현재 월 강조 표시** — 지금이 몇 월인지 색/배지로 분명히 눈에 띄게. (가능하면 화면 열 때 현재 월로 자동 스크롤)
3. **가독성 개선** — 지금 빽빽하고 보기 힘든 부분을:
   - 월 단위 구획을 명확히 나누고 (구분선·여백)
   - 글자 대비·크기 확보해 한눈에 읽히게
   - 중요한 정보(월·핵심 진도)가 먼저 보이도록 위계 정리

> 목표 그림: "달력을 펼치듯, 올해 흐름이 한눈에. 지금 달은 형광펜 친 것처럼 도드라지게."

## 6. [Phase 5] 화면별 최종 점검 (모든 화면에 같은 체크리스트 적용)

위 작업을 마친 뒤, **화면을 하나씩** 열어 아래 7가지를 점검·수정한다. 한 화면이라도 빠뜨리지 않는다.

```
[ ] 1. 같은 기능에 같은 부품을 쓰고 있나? (다른 화면과 비교)
[ ] 2. 색·간격·글자가 전부 토큰인가? (직접 박은 값 0개)
[ ] 3. 글자 대비 4.5:1 이상인가?
[ ] 4. 누르는 영역이 44×44 이상인가?
[ ] 5. 로딩 / 비었을 때 / 오류 상태가 다 있나?
[ ] 6. 문구가 다른 화면과 같은 단어를 쓰나?
[ ] 7. 데이터가 나중에 들어와도 화면이 안 밀리나?
```

대상 화면(빠짐없이): 첫 화면(오늘/대시보드), 초등수학 보강관리, 중고등 보강관리, 초등 보강관리, 출결, 숙제, 진도, 수학 연간 수업 계획, 그 외 모든 화면.

## 7. ⚠️ 절대 지킬 주의사항 (건드리면 안 되는 것)

1. **노션 동기화 규칙을 깨지 말 것.** 데이터 종류별로 방향이 정해져 있다:
   - 학생 명단: 노션 → 앱 (한 방향만)
   - 수업/출결 기록: 앱 → 노션 (한 방향만)
   - 이 방향을 절대 바꾸거나 양방향으로 만들지 말 것.
2. **기능 로직은 손대지 않는다.** 이번 작업은 *디자인·성능·보기 편함*만. 계산·저장·연동 로직은 그대로.
3. **한 번에 한 화면씩**, 작업 후 화면이 안 깨졌는지 확인하고 다음으로 넘어간다.
4. 기존 데이터·연동·접속 주소는 그대로 유지한다.

## 8. 작업 순서 (이 순서대로)

```
1) 전체 화면 목록 + 부품 점검표 작성 (진단)
2) 토큰 표 정리 + 하드코딩 값 토큰으로 교체
3) 공통 부품(버튼·카드·탭·리스트·모달 등) 하나로 통일
4) 보강관리 3종 '입는 옷' 통일
5) [확인 후] 홈 결정 → 첫 화면 개편
6) 성능 개선 (캐싱·스켈레톤·자리고정·낙관적 업데이트·부드러운 전환)
7) 수학 연간 수업 계획 (연도별 보기·현재 월 강조·가독성)
8) 화면별 7가지 체크리스트 최종 점검
```

## 9. Claude Code에게 전달할 시작 명령 (복붙용)

```
첨부한 '바꿈 수업관리 통일·마감 업데이트 지시서'대로 작업해줘.

먼저 Phase 1의 "전체 화면 목록 + 부품 점검표"부터 만들어서 나한테 보여줘.
어느 화면이 어떻게 다른지, 하드코딩된 값이 어디에 있는지 표로 정리해줘.
그걸 같이 본 다음에 토큰 정리부터 순서대로 진행하자.

주의: 노션 동기화 방향(학생명단 노션→앱, 기록 앱→노션)과 기능 로직은
절대 건드리지 말 것. 디자인·성능·가독성만 다듬는다.
한 번에 한 화면씩, 깨지는지 확인하며 진행.
```
