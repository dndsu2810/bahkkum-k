# 바꿈 통합 허브 — 작업지시서 (역할 기반)

**대상:** 바꿈영수학원 전체 운영을 묶는 **통합 허브** 신규 구축.
**한 줄 요약:** 로그인하면 역할에 맞는 **사이드바가 바로 열린다.** 수학 앱 기존 메뉴는 "수학 수업관리" 카테고리 안으로, 영어도 같은 형태로 구성. 학생·특이사항·업무보드는 공유.
**관계 문서:** 본 문서가 「영어 과목 추가 작업지시서」의 구조 부분을 대체·확장한다. 화면 폴리시·봇·다크모드는 「v3 통합 업그레이드」 참고.

---

## 디자인 시스템 & 핸드오프 (깔끔·단순)
**원칙:** 한 디자인 언어로 통일 · 역할 화면마다 꼭 필요한 것만 · 복잡함/과밀 금지 · 다크모드 포함. 화면 폴리시 세부는 「v3」, 다크 토큰은 「다크모드 핸드오프」를 따른다.

### 공통 컴포넌트 (전 역할 동일하게)
- **사이드바 로고 아래 이름·담당 표시:** 로고/학원명 바로 아래에 `(이름)님, 담당: (과목/역할)` 한 줄. 예: `이지현님, 담당: 수학 · 원장` / `(이름)님, 담당: 영어 중고등`. 글자 크기 12~13px, 보조 색(#64748B). 로그인 계정 정보에서 자동으로 채워짐.
- **카드:** 통계·목록 한 규격. hover 시 살짝 떠오름(클릭 가능 신호).
- **배지·태그:** 학년·상태·과목 한 세트(색·모양·크기 통일).
- **버튼/상태:** 선택된 상태만 색 채움, 나머지는 회색.
- **빈 상태/로딩:** 친절한 한 줄 + 스켈레톤.
- **토스트+되돌리기:** 입력 후 피드백.

### 역할 화면 — 단순함 규칙
- 각 화면 첫 진입엔 그 역할의 **핵심 한두 가지만**. 세부는 펼쳐보기.
- **원장:** 전체를 보되 요약 먼저.
- **데스크:** 표(전체 시간표·리스트) 중심, 조회 편의.
- **영어:** 인센티브·복잡 지표 없이 목표·숙제·코멘트 중심.
- **학생:** 최소·읽기 위주, 군더더기 없이.

### 내비게이션 · 반응형 · 접근성
- 허브 로그인 → 역할 화면. 상단/사이드에 역할·과목 표시. 얕고 명확한 메뉴.
- 데스크탑(사이드바) / 태블릿(축소) / 모바일(하단 탭바). 터치 영역 44px.
- 명암비 4.5:1↑ · 키보드 포커스 표시 · 화면별 h1 · 색에만 의존 금지(라벨 병행).

## 1. 전체 구조 — 핵심 개념

**핵심:** 허브는 별도 화면이 아니다. **로그인하면 그 사람의 사이드바가 바로 열린다.**
수학 앱을 왔다갔다 하는 구조가 아니라, **역할에 맞는 카테고리들이 사이드바에 구성**되어 있는 것.
강사에게 화면을 배분한다는 것 = 원장이 그 강사의 **사이드바에 보일 카테고리/메뉴를 지정**하는 것.

### 역할별 사이드바 구성

**이지현 (원장 + 수학 전체)**
```

이지현님, 담당: 수학 · 원장       ← 로고 아래 이름 + 담당 표시
├── 홈
├── 강사 업무 보드
│
├── [수학 수업관리]        ← 기존 앱 메뉴 그대로 이 카테고리 안으로
│   ├── 오늘
│   ├── 시간표
│   ├── 대시보드
│   ├── 학원 일정
│   ├── ─ 수업 기록
│   ├── 출결 기록
│   ├── 숙제 기록
│   ├── 진도 기록
│   ├── 테스트 기록
│   ├── ─ 관리
│   ├── 학생 관리
│   ├── 보강 관리
│   └── 수학 월말리포트    ← 수학 전용
│
├── [공통]
│   ├── 강사 특이사항
│   ├── 바꿈 매뉴얼
│   └── SNS 관리
│
└── [원장 전용]
    ├── 학생 마스터
    ├── 영어 월말리포트    ← 영어 전용 (원장이 관리)
    ├── 강사 관리 (등록·배분)
    └── 설정
```

**영어 강사 (중고등)**
```
바꿈영수학원
(이름)님, 담당: 영어 중고등        ← 로고 아래 이름 + 담당 표시
├── 홈
├── 강사 업무 보드
│
├── [영어 수업관리 — 중고등]  ← 수학과 같은 형태, 영어 특성만 다름
│   ├── 오늘
│   ├── 시간표
│   ├── 대시보드
│   ├── 학원 일정
│   ├── ─ 수업 기록
│   ├── 출결 기록

│   ├── 진도 기록
│   ├── 테스트 기록
│   ├── ─ 관리
│   ├── 학생 관리
│   └── 보강 관리
│
└── [공통]
    ├── 강사 특이사항
    └── 바꿈 매뉴얼
```

**초등 영어 강사** — 위와 동일 구성, `(이름)님, 담당: 영어 초등`으로 표시

**데스크쌤**
```
바꿈영수학원
(이름)님, 담당: 데스크             ← 로고 아래 이름 + 담당 표시
├── 홈
├── 전체 시간표
├── 학생 정보
```

**학생** — 내 학습 기록 조회 전용

### 원칙
- 로그인하면 곧바로 해당 역할 사이드바 (별도 "허브 화면" 이동 없음)
- 수학 앱 기존 메뉴는 구조 변경 없이 **"수학 수업관리" 카테고리 안으로** 들어감
- 영어도 **수학과 같은 형태**로 카테고리 구성 (항목만 영어 특성에 맞게)
- 공통 메뉴(특이사항·매뉴얼·업무 보드)는 **학생 제외 전원** 사이드바에 표시

## 2. 역할 · 권한
> 지현님이 제작자이므로 **지현님이 원장(관리자) 권한 보유** + 수업 역할은 **수학 전체(초등+중고등)**. (수학은 지현님 1인 통합 담당, 영어만 초등/중고등 강사 분리)

| 역할 | 기본 화면 | 볼 수 있는 것 | 할 수 있는 것 |
|---|---|---|---|
| **원장(관리자)** | 전체 | 모든 화면·전 과목 | **강사 등록·배분**, 전 과목 열람, 설정 |
| **수학 강사**(지현) | 수학(초등+중고등 전체) | 담당 수학 학생 전체 | 출결·숙제·진도·테스트 입력 |
| **영어 강사** | 영어(중고등) | 담당 중고등 영어 학생 | 영어 일일기록·테스트 입력 |
| **초등영어 강사** | 영어(초등) | 담당 초등 영어 학생 | 영어 일일기록 입력 |
| **데스크쌤** | 데스크 | 전체 시간표·학생 정보·강사 계정 리스트 | 안내·조회(운영 보조) |
| **학생** | 학생 화면 | 본인 기록(목표·숙제·진도 등) | 조회(필요 시 제출) |

- **공통:** 누가 로그인하든 **강사 특이사항·업무 보드**는 함께 본다(아래 7·8).
- 원장은 강사 계정을 **등록**하고 담당(과목/학년)을 **배분**한다.
- **매뉴얼 위키:** 학생을 제외한 전원(원장·수학·영어·초등영어·데스크)이 열람(아래 10).

## 3. 공통 학생 마스터 (공유)
- 한 학생당 한 번만 등록. 노션 학생 DB 구조를 그대로 따른다.
- 항목: 이름 · 온라인ID · 생년월일(→자동 비번) · 학교 · 학년(자동) · 첫수업일 · 학부모/학생 연락처 · 상태(재원/휴원/퇴원/대기) · 생일(자동) · 적립/포인트.
- 한 학생이 수학·영어를 모두 들으면 **같은 학생에 양쪽 기록**이 붙는다.

## 4. 수학 (기존 그대로)
- 지현님의 현재 앱(bakkum-class)을 **그대로 유지**. 허브에서 '수학 강사' 화면으로 연결만.
- 화면 개선(글씨 대비·상태색·오늘 브리핑 등)은 「v3」 적용.

## 5. 영어 (신규) — 영어 특성 반영 ⭐
구성은 수학과 **유사하되 영어에 맞게** 만든다.

| 항목 | 영어 사양 |
|---|---|
| 학년 구분 | **초등 영어 / 중고등 영어 분리** (강사도 분리 담당) |
| 일일 기록 | **학습 목표(체크) · 숙제(자유 입력) · 숙제검사 · 코멘트 · 자료 배부** (학습일지 방식) |
| 진도/커리 | 교재명+레벨 자유 입력 (예: Core Phonics 4 / Insight Link L1 / 모의고사) |
| 테스트 | **수학과 성격 다름** — 단어시험 점수 등 영어식 평가. (경시·랭킹식 아님) |
| 대시보드 | **인센티브제 없음** → 수학 대시보드의 인센티브 정산·규칙 **제외**. 영어는 등원/진도/숙제 현황 위주 |
| 출결 | **수학과 약간 다름** — 학습일지처럼 '오늘 등원 학생 선택' 중심(수학은 시간표 기반 출석/지각/결석). 세부 차이는 원장님 확인 |
| 저장 | **앱(허브)에 자체 저장** — 영어는 노션에 따로 저장하지 않음 |

> ⚠️ 영어를 수학 복제로 만들지 말 것. 위 표대로 **인센티브 제거 · 테스트 성격 변경 · 출결 방식 조정 · 초등/중고등 분리**를 반드시 반영.

## 6. 카카오워크 봇 (역할별) — ⏸️ 보류 (가장 마지막)
> **현재 미확정. 우선순위 최하위로 미룬다.** 아래는 방향 기록일 뿐 확정 아님.
- 동작(예정): v3 일일 브리핑 봇을 **역할별로 데이터만 걸러** 각자 1:1 발송.
  - 원장 → 전체(수학+영어) · 수학 담당 → 수학만 · 영어 담당 → 담당 영어만 · 데스크 → 운영 브리핑(여부 미정)
  - 구조·시간·문구·생일은 「v3 PART D」 그대로, '누구 데이터냐'만 역할로 분기.
- **확정 필요:** 받는 사람, 데스크 포함 여부, 영어 봇 필요 여부, 발송 시간.
- **작업 시점:** 다른 기능을 다 만든 **뒤 맨 마지막**.

## 7. 강사 특이사항 (공용 누적)
- 강사들이 학생별 특이사항을 기록하던 도구를 **허브의 공용 기록**으로 흡수.
- **목적:** 원장님이 전반을 관찰. → **데이터가 잘 누적되고, 모두가 공유·열람** 가능해야 한다.
- 학생별로 메모가 시간순 누적(누가·언제·무엇). 과목 공통(수학·영어 가리지 않음).
- *완료 기준:* 어떤 강사가 남긴 특이사항도 모두가 학생별로 모아 볼 수 있고, 기록이 사라지지 않고 쌓인다.

## 8. 강사 업무 보드 (공유 · 실시간)
- **변경:** 이전 '1인 전용'에서 → **모든 선생님이 공유, 실시간으로 함께** 보는 보드로 한다. (이 결정이 v3 PART E의 1인 설정을 대체)
- 칸반 3칸(할 일/진행 중/완료), 드래그 이동, 완료 7일 후 자동 보관 + 보관함 조회(되살리기)는 v3 그대로.
- 누가 카드를 옮기면 **다른 선생님 화면에도 곧 반영**(실시간 공유).
- 결석→보강 자동 카드도 유지(담당 과목 결석 기준).

## 9. 데스크쌤 화면
- **전체 학생 시간표**(수학·영어 통합 보기)
- **강사 계정 리스트**(로그인 아이디/비밀번호 등 운영 정보)
- **학생 정보** 조회(연락처·상태 등)
- 성격: 운영·안내 보조용 조회 중심. (민감 정보는 원장이 권한 관리)

## 10. 바꿈 매뉴얼 (위키) — 학생 제외 전원 공유
- 노션 「바꿈 매뉴얼 보기」를 허브 안의 **위키**로 제공. **강사·데스크가 모두 열람**(학생만 제외).
- 글마다 **제목 · 중요도(낮음~핵심) · 상태(초안/작성중/검토중/최신/업데이트 필요) · 최종 수정일** 표시. 목록(테이블)에서 중요도·상태로 정렬·필터.
- 운영 방식: 노션 매뉴얼을 **보기 동기화(읽기 전용)** 하거나 허브로 이관. 어느 쪽이든 **학생 화면엔 노출 안 함**.
- *완료 기준:* 강사·데스크가 매뉴얼 글을 목록·본문으로 보고, 학생에겐 보이지 않는다.

## 11. 월말리포트 — 이미지 일괄 생성 (수학·영어 완전 분리)
- **수학 월말리포트:** 수학 수업관리 카테고리 안에 위치. 수학 담당(지현)이 사용.
- **영어 월말리포트:** 원장 전용 카테고리 안에 위치(영어 데이터 취합·이미지 생성은 원장 권한). 초등/중고등 탭으로 구분.
- 두 리포트는 **완전히 분리**된 메뉴로 존재. 사이드바에서 뒤섞이지 않음.
- 현재 방식: 노션 데이터를 성적표 사이트에 넣어 이미지로 저장. 수학은 이미 운영 중, **초등영어도 같은 구조**(성적표 시스템: `dndsu2810.github.io/bahkkum-k/english-report/dist`).
- 영어 데이터 출처: 「초등 리포트 DB」 — 학생별 월 평가 8개 항목(**Listening · Speaking · Reading · Spelling+Writing · Comprehension · Task Performance · Learning Attitude · Confidence**, 등급 P~NI) + Comments + 평가월 + 담당T.
- **개선(핵심):** 지금은 'JSON 출력'을 **한 명씩 복붙**해 이미지를 뽑음 → 한 달치 **전체 학생을 한 번에 불러와 이미지 일괄 저장**(학생명 파일명, 필요 시 zip). 수학 월말리포트와 동일 흐름.
- *완료 기준:* 수학·영어 월말리포트가 각자 위치에서 분리 운영. 월·반 선택 → 해당 학생 전원 성적표 이미지 **일괄 저장**(한 명씩 복붙 불필요).

## 12. SNS 관리 — 등록 → 데스크 복붙 업로드
- **열람:** **데스크·강사 모두**(노션 'SNS 관리'처럼 운영 중).
- **흐름:** ① 강사/원장이 올릴 글 **등록**(제목 · 본문 · 대상 SNS · 작성자) → ② **데스크가 본문을 복붙해 업로드** → ③ 상태 '업로드 완료' + **업로드 링크** 기록.
- **상태값**(노션 그대로): 업로드 대기 · 수정필요 · 업로드 중지 · 업로드 완료.
- (선택) 대상 채널 구분: 블로그 · 인스타 등.
- *완료 기준:* 등록된 글을 데스크가 목록에서 보고, 본문 복사 → 업로드 → 완료 표시까지 된다.

## 작업 순서 (로드맵)
1. **허브 + 로그인 + 역할 라우팅** — 역할별로 다른 화면이 뜨게(원장/수학/영어/초등영어/데스크/학생).
2. **공통 학생 마스터** — 노션 구조 기반, 수학·영어 공유.
3. **수학 연결** — 기존 앱을 '수학 강사' 화면으로 붙이기(기능 그대로).
4. **영어(신규)** — 5번 표대로(초등/중고등 분리·인센티브 제거·테스트/출결 조정·일일기록).
5. **공용 특이사항** — 학생별 누적·전체 공유.
6. **공유 업무 보드** — 실시간 공유로.
7. **데스크 화면** — 전체 시간표·계정 리스트·학생 정보.
8. **매뉴얼 위키** — 학생 제외 전원 열람(노션 동기화 또는 이관).
9. **월말리포트 이미지 일괄 저장**(수학·영어).
10. **SNS 관리**(등록 → 데스크 복붙 업로드).
11. 각 단계 **기존 수학 기능 회귀 확인**.
12. **(보류) 카카오워크 봇 역할별** — 확정 후 가장 마지막에.

## 확인 필요 (원장님/현장과)
- 영어 **출결 방식**의 정확한 차이(등원 선택 vs 출석/지각/결석)
- 영어 **테스트 기록 방식**(단어시험 만점·회차 등)
- 데스크쌤이 **비밀번호 리스트**까지 보는 게 맞는지(보안 범위)
- 강사 계정·담당 **배분 규칙**(한 강사가 여러 과목/학년 가능 여부)
