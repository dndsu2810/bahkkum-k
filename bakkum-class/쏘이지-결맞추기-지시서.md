특히 이번 판에서는 **말투(UX 라이팅)** 를 토스 스타일로 또렷하게 정돈하는 데 무게를 뒀어요. (뒤쪽 '부록. 쏘이지 말투 사전' 참고)

## 작업 전에 꼭 지켜주세요 (안전 수칙)

아래는 절대 건드리면 안 됩니다. 이건 "디자인 정돈" 작업이지 "구조 변경" 작업이 아닙니다.

1. **기존 기능을 없애지 마세요.** 화면·버튼·항목을 삭제하지 말고, 모양과 말투만 다듬습니다.
2. **노션 연동 방향을 절대 바꾸지 마세요.** (학생 명단: 노션 → 앱 / 출결·숙제·진도·테스트 기록: 앱 → 노션). 이 방향은 손대지 않습니다.
3. **데이터 저장 경로를 바꾸지 마세요.** 수학(로컬 스토어)과 영어(워커 API)는 저장되는 곳이 완전히 다릅니다. 한쪽을 고친다고 다른 쪽을 같이 고치지 마세요.
4. **이미 저장된 데이터의 모양(구조)을 바꾸지 마세요.** 살아 있는 기록이 깨질 수 있습니다.
5. **한 번에 한 화면씩** 작업하고, 끝나면 멈춰서 확인받습니다. 새 색·새 부품(컴포넌트)을 새로 만들지 말고, **이미 있는 것**을 가져다 씁니다.

# 1부. 브랜드 결 통일 (가장 먼저)

## 1-1. 상단 로고와 학원명 표기를 쏘이지로 통일

- **화면**: 모든 화면 왼쪽 위 로고 영역 + 왼쪽 메뉴 맨 아래
- **지금 모습**: 왼쪽 위에는 **청록색(어두운 초록) '바꿈영수학원' 옛 로고**가 있고, 왼쪽 메뉴 맨 아래에는 **노란 꿀벌 'Soez' 로고**가 따로 있습니다. 한 화면에 옛 브랜드(청록)와 새 브랜드(꿀색)가 동시에 보입니다.
- **바꿀 동작**: 위·아래 로고를 **쏘이지(꿀색·벌) 한 가지로 통일**합니다.
  - (결정 필요) 새 로고 이미지가 아직 확정 전이라면, **지금은 텍스트 표기만 '쏘이지'로 맞추고 색을 꿀색 계열로** 정돈해 두고, 로고 이미지 교체는 마스코트 확정 후에 합니다.
- **다 됐다고 볼 기준**: 한 화면 안에서 청록색 옛 로고와 꿀색 새 로고가 **동시에 보이지 않습니다.**

## 1-2. 모든 문구를 쏘이지 말투(해요체)로 통일

- **화면**: 모든 화면의 제목 아래 설명문, 빈칸 안내(placeholder), 비어 있을 때 안내문, 알림·완료·오류 문구
- **지금 모습**: 같은 종류의 문장인데 말투가 섞여 있습니다.
  - '하세요'체: "빠르게 입력하세요", "출결·태도·숙제를 기록하세요"
  - '해요'체: "순위예요", "한눈에 봐요", "검색해서 넣어요"
- **바꿀 동작**: 화면의 모든 말을 **부드러운 해요체로 통일**합니다. 단순히 어미만 바꾸는 게 아니라, 뒤쪽 **'부록. 쏘이지 말투 사전'의 5가지 원칙과 교정표**를 기준으로 고칩니다.
  - 핵심: 군더더기 빼기 / 강요 대신 권유 / 한자어를 동사로 풀기 / 끝낸 순간엔 따뜻한 한마디.
  - 단, **버튼 글자는 짧은 동사형 유지**합니다. ('저장', '추가', '전체 출석')
- **다 됐다고 볼 기준**: 화면 설명·빈칸 안내·빈 상태·완료·오류 문구의 말투가 **전부 해요체로 같아지고**, 부록 교정표의 'Do' 문장과 결이 맞습니다.

## 1-3. 학생 아바타 모양을 육각형(벌집)으로 통일

- **화면**: 학생 얼굴 대신 들어가는 도형(아바타)이 나오는 모든 곳 (포인트 랭킹, 학생 명단, 카드 등)
- **지금 모습**: 포인트 랭킹은 **육각형(벌집)** 아바타로 잘 되어 있습니다. 다른 화면은 모양이 다를 수 있어, 같은 학생이 화면마다 다른 도형으로 보일 수 있습니다.
- **바꿀 동작**: 학생 아바타를 **어디서나 같은 육각형(벌집)** 한 모양으로 맞춥니다. (쏘이지 = 벌·벌집 세계관과 맞춤)
- **다 됐다고 볼 기준**: 어느 화면에서 봐도 학생 아바타가 **같은 육각형 모양**입니다.

# 2부. 화면 다듬기 (UI)

## 2-1. 출결 버튼을 '눌러야 하는 버튼'으로 보이게

- **화면**: 수학 '오늘' · '대시보드', 영어의 같은 화면 안 출결 영역
- **지금 모습**: '출석·지각·결석·조퇴·무단결석'이 **연한 회색 바탕에 글자만** 놓여 있어 눌러야 하는 버튼인지 잘 모릅니다. 특히 **다크모드(어두운 화면)에선 글자가 배경에 묻혀 거의 안 보입니다.**
- **바꿀 동작**:
  - 다섯 항목을 **또렷한 버튼 모양**(테두리 또는 옅은 배경 + 진한 글자)으로 만듭니다.
  - 선택된 항목은 **뜻에 맞는 색**으로 채워 한눈에 구분되게 합니다. (출석=초록 / 지각=노랑 / 결석·무단결석=빨강 / 조퇴=주황 — 이미 쓰는 색 안에서)
  - **다크모드에서도 글자가 또렷하게** 보이도록 합니다.
- **다 됐다고 볼 기준**: 밝은 화면·어두운 화면 모두에서 **다섯 버튼이 또렷이 보이고, 무엇을 골랐는지 색으로 바로 구분**됩니다.

## 2-2. 강사 정보 입력칸 잘림 해결

- **화면**: 공통 > 강사 정보 안내
- **지금 모습**: 이름칸이 좁아 **'정민아(원장님)'이 '정민아(원장능'처럼 잘려** 보이고, **전화번호 뒷자리도 잘립니다.**
- **바꿀 동작**: 이름·연락처 칸의 폭을 넓혀 내용이 안 잘리게 합니다. 화면이 좁아지면 **칸을 아래로 내려(줄바꿈) 전부 보이게** 합니다.
- **다 됐다고 볼 기준**: 긴 이름과 전화번호가 **잘리지 않고 끝까지** 보입니다.

## 2-3. '보강' 글자와 버튼이 겹치는 것 정리

- **화면**: '오늘' · '대시보드' 카드의 보강 영역
- **지금 모습**: '보강'이라는 작은 제목 글자 위로 **'보강 완료' 버튼이 살짝 겹쳐** 글자가 가려집니다.
- **바꿀 동작**: 제목 글자와 버튼 사이에 **세로 간격을 더 줍니다.**
- **다 됐다고 볼 기준**: '보강' 글자가 버튼에 **가려지지 않습니다.**

## 2-4. 포인트 랭킹의 'N일' 숫자가 무슨 뜻인지 보이게

- **화면**: 공통 > 포인트 랭킹
- **지금 모습**: 학생 이름 옆에 '135일', '30일' 같은 숫자가 있는데 **무슨 기간인지 알려주는 말이 없습니다.**
- **바꿀 동작**: 숫자 앞이나 뒤에 **의미를 붙입니다.** (예: '함께한 지 135일' 또는 '재원 135일')
  - (결정 필요) 이 숫자가 정확히 무엇을 뜻하는지 지현님이 알려주시면 그 단어로 적습니다.
- **다 됐다고 볼 기준**: 숫자만 봐도 **무슨 기간인지** 알 수 있습니다.

# 3부. 더 편하게 (UX · 모바일 · 접근성)

## 3-1. 휴대폰에서 메뉴 열기 쉽게

- **화면**: 휴대폰(좁은 폭) 전체
- **지금 모습**: 휴대폰에서는 왼쪽 메뉴가 접히는데, **메뉴를 여는 버튼(가로 세 줄 모양)이 보이지 않아** 다른 화면으로 가기가 어렵습니다. 로고나 제목을 눌러도 열리지 않습니다.
- **바꿀 동작**: 휴대폰 화면 **왼쪽 위에 또렷한 '메뉴 열기' 버튼을 항상** 띄워두고, 누르면 메뉴가 옆에서 미끄러져 나오게 합니다.
- **다 됐다고 볼 기준**: 휴대폰에서 **어느 화면에 있든 한 번 눌러 메뉴를 열 수** 있습니다.

## 3-2. 휴대폰에서 출결 버튼 누르기 쉽게

- **화면**: 휴대폰의 출결 영역
- **지금 모습**: 다섯 버튼이 **한 줄에 빽빽하게** 붙어 있어 손가락으로 정확히 누르기 어렵습니다.
- **바꿀 동작**: 버튼의 높이와 간격을 **손가락에 맞게(한 변 최소 44픽셀)** 키우고, 한 줄이 좁으면 **두 줄로** 자연스럽게 내립니다.
- **다 됐다고 볼 기준**: 휴대폰에서 출결 다섯 개를 **헷갈리지 않고 정확히** 누를 수 있습니다.

## 3-3. 빈 상태·불러오는 중·오류 안내를 똑같은 결로

- **화면**: 데이터가 없을 때 / 불러오는 중일 때 / 오류가 났을 때 (모든 화면)
- **지금 모습**: 어떤 화면은 친절한 빈 상태 문구와 불러오는 중 표시가 있고, 어떤 화면은 **휑하거나 멈춘 것처럼** 보입니다.
- **바꿀 동작**: 세 상황의 안내를 **하나의 결로** 맞춥니다. (말투는 부록의 '상황별 문구' 기준)
  - 비었을 때: "아직 없어요" + **어떻게 시작하는지 한 줄**
  - 불러오는 중: 이미 쓰는 **회색 깜빡임(스켈레톤)** 으로 통일
  - 오류일 때: **무엇이 / 어떻게** 한 줄 안내 (에러 코드·영어 그대로 노출 금지)
- **다 됐다고 볼 기준**: 세 상황(없음·로딩·오류)의 안내가 **모든 화면에서 비슷한 모양과 말투**입니다.

# 부록. 쏘이지 말투 사전 (UX 라이팅 가이드)

> 이 부록은 1-2(말투 통일)와 3-3(빈 상태·오류)을 작업할 때 보는 **기준표**입니다.
> 토스의 라이팅 원칙을 쏘이지 상황에 맞게 옮겼어요. 클코는 화면 문구를 고칠 때 아래 'Do' 쪽 결을 따릅니다.

## 쏘이지의 목소리 (한 줄)

쏘이지는 **바쁜 선생님 곁에서 군더더기 없이 거드는, 다정한 조수**처럼 말해요.
빠르게 일하는 선생님에겐 짧고 분명하게, 학생에겐 따뜻하게 건넵니다.

## 다섯 가지 원칙

1. **명확하게** — 무슨 일이 일어나는지 바로 알 수 있게. 모호한 '확인·제출' 대신 **결과를 적어요.** (예: '저장' 대신 '리포트 저장')
2. **간결하게** — 빼도 뜻이 통하는 군더더기(잡초)는 잘라요. (예: "앞으로 더 나은 서비스를 제공하겠습니다" → "더 나은 기록을 도와드릴게요")
3. **친근하게** — 격식체(하십시오체)를 걷어내고 **해요체**로. (예: "진행하십시오" → "진행해 주세요")
4. **존중하게** — 겁주거나 강요하지 않고 **권유**해요. 최종 선택은 사용자 몫. (예: "삭제하면 복구 불가" → "지우면 되돌릴 수 없어요. 그래도 지울까요?")
5. **공감하게** — 무언가를 끝낸 순간엔 **따뜻한 한마디**를 더해요. (특히 학생 화면)

## 교정표 ③ — 버튼·완료 안내 (CTA와 끝낸 순간)

버튼은 **무엇을 하는지** 결과로 적고, 끝낸 순간엔 **공감 한마디**를 더해요.

| 자리 | 지금 (Don't) | 이렇게 (Do) | 적용 원칙 |
| --- | --- | --- | --- |
| 추가 버튼 | 추가 | 시험 추가 / 학생 추가 (무엇을 추가하는지) | 명확하게 |
| 전체 출석 누른 뒤 | (반응 약함) | 오늘 등원한 학생을 모두 출석 처리했어요 | 명확하게, 공감하게 |
| 월말리포트 저장 | 저장됨 | 리포트를 저장했어요 (여러 장이면 'N장을 저장했어요') | 명확하게 |
| 학생 화면, 숙제 다 체크 | (없음) | 오늘 숙제를 모두 끝냈어요. 잘했어요! | 공감하게 |

## 교정표 ④ — 오류 메시지 (막힌 곳을 다음으로 안내)

영어·에러 코드를 그대로 보여주지 않아요. **무슨 일인지 + 어떻게 하면 되는지** 두 가지만 담아요.

| 상황 | 지금 (Don't) | 이렇게 (Do) |
| --- | --- | --- |
| 불러오기 실패 | 불러오기 실패 / Error | 기록을 못 불러왔어요. 잠시 후 다시 시도해 주세요 |
| 저장 실패 | 저장 실패 | 방금 입력한 내용을 저장하지 못했어요. 다시 한 번 저장해 주세요 |
| 권한이 없는 설정 저장 | (조용히 실패 / 반응 없음) | 이 설정은 원장님 계정에서 바꿀 수 있어요 |
| 네트워크 오류 | Network timeout (500) | 인터넷 연결이 불안정해요. 잠시 후 다시 시도해 주세요 |

> 마지막 항목('권한이 없는 설정 저장')은 지금 일부 화면에서 **저장이 조용히 실패**할 수 있어요. 사용자가 "저장이 됐나?" 헷갈리지 않게, 위 'Do' 문구를 띄워주면 좋아요.

# 지현님이 정해주실 것 (결정 포인트)

진행하기 전에 아래 세 가지만 정해 주시면 나머지는 그대로 진행할 수 있어요.

1. **로고 교체 시점** (1-1)
   - (가) 지금 바로 텍스트·색만 쏘이지로 맞추고, 로고 이미지는 마스코트 확정 후 교체 — **추천**
   - (나) 마스코트 확정까지 기다렸다가 한 번에 교체
2. **출결 선택 색** (2-1)
   - 출석=초록 / 지각=노랑 / 결석·무단결석=빨강 / 조퇴=주황 으로 진행해도 될지
3. **포인트 랭킹 'N일'의 뜻** (2-4)
   - '재원 일수'인지, '마지막으로 꿀 받은 날로부터의 일수'인지, 다른 뜻인지
