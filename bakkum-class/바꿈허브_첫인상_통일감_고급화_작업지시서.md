# 바꿈 허브 — 첫인상·통일감·고급화 종합 작업지시서

**검토:** 정민아/이지현 원장으로 전 화면 점검(2026-06-15). 로그인·홈·설정·일정·오늘 등 정밀 확인.
> 🎯 **목표:** 첫 입장부터 끝까지 **깔끔·통일·고급**. "다 따로 노는 느낌"을 없애고, 디테일 한 끗으로 전문적인 어드민이 되게.
> ⚠️ **수학 데이터 불변**(화면·디자인은 개선 OK).

## ✅ 이미 잘 된 것(확인)
로고 적용(설정에서 업로드·크기조절) · 원장 "담당 과목" 표기 빠짐 · 전역 학생 검색 · 헤더 알림(🔔) · 영어 그룹명 괄호 통일 · 시간표 변경 승인 배너 동작 · 설정에 "노션 기록 1회 가져오기" 존재 · 오늘 요약 줄.

---

## 1. 스플래시(첫 진입) 화면 ⭐
- 앱 첫 진입·**로딩/대기·화면 전환에 시간이 걸릴 때** 보이는 스플래시.
- **보낸 ez 손그림 로고 이미지** 중앙 배치 + 학원명("바꿈영수학원") + 살짝의 페이드/로딩 인디케이터.
- 깔끔한 단색(브랜드 네이비/슬레이트) 배경, 군더더기 없이.
- 너무 길게 잡지 말고(로딩 끝나면 바로 사라짐), 렉·전환 시 흰 화면 대신 이게 보이게.
- *완료 기준:* 입장/로딩/전환 때 흰 깜빡임 대신 ez 로고 스플래시가 잠깐 보인다.

## 2. 로그인 인사 문구 ⭐
- 현재 로그인 카드 제목이 "바꿈 통합 허브"라 딱딱함.
- **첫 입장 느낌의 인사**로: 상단에 `Hi! 바꿈영수학원` (또는 `안녕하세요 :) 바꿈영수학원`) + 아래 작게 "선생님·학생 로그인".
- 하단 "제작자 EZ"는 유지하되 톤 정리(작게, 은은하게).
- *완료 기준:* 로그인 화면이 환영받는 첫인상으로 보인다.

## 3. 전반 UI/UX 통일 (다 따로 노는 것 정리) ⭐
- **날짜 형식 1가지:** `2026년 6월 15일 (월)`로 통일(네이티브 날짜칸 `2026.06.17.` 같은 것 → 앱 공통 컨트롤).
- **날짜 이동 컨트롤 통일:** 모든 화면 `‹ 이전 · 오늘 · 다음 ›` 같은 모양으로.
- **카드·여백·모서리·그림자** 한 규격(섹션 카드 동일 radius·padding·border).
- **버튼 위계 통일:** 주요(파랑 채움)·보조(외곽선)·위험(빨강) 3종만, 위치·크기 일관.
- **배지·칩**(초등/중등, 출석/지각/결석, 상태) 색·크기·모양 통일.
- **빈 상태** 문구+행동버튼 패턴 통일.
- *완료 기준:* 어느 화면을 가도 같은 디자인 언어로 보인다(따로 노는 느낌 제거).

## 4. 노션 동기화 — 설정에 버튼 하나로 통합 ⭐
- **문제:** 동기화/가져오기 버튼이 흩어짐 — 설정의 "노션 기록 1회 가져오기" + 학원 일정의 "노션 일정 가져오기"가 따로.
- **수정:** **설정에 '노션에서 가져오기' 버튼 1개만** 남기고, 다른 화면의 가져오기 버튼은 **모두 제거**.
- 그 버튼 1개를 누르면 — **이미 있는 정보는 건너뛰고**, 노션에서 끌어오려던 **모든 연결 항목을 한 번에** 가져옴: 학생명단·출결·숙제·진도·테스트·**학원 일정**·할 일 배정·생일 등.
- **중복 방지:** 이미 들어온 건 다시 안 들어옴(추가된 것만). 진행 표시(가져오는 중/완료 n건).
- 문구도 명확히: "노션에 쌓인 기록을 앱으로 가져옵니다. 이미 있는 건 건너뜁니다."
- *완료 기준:* 설정의 버튼 1개로 노션의 모든 연결이 중복 없이 넘어오고, 다른 화면엔 가져오기 버튼이 없다.

## 5. 고급·전문 디테일 (한 끗 차이) ⭐
- **타이포:** 제목/부제/본문 크기·굵기·자간 위계 명확히(들쭉날쭉 제거).
- **색 대비:** 회색 글씨가 너무 흐린 곳 가독성 보정.
- **정렬:** 숫자·금액·시간 오른쪽 정렬, 라벨↔값 그리드 정렬.
- **마이크로 인터랙션:** 버튼 호버/클릭, 탭 전환, 토스트(저장됨) 부드럽게(과하지 않게).
- **로딩 스켈레톤:** 목록 불러올 때 빈 화면 대신 회색 스켈레톤.
- **아이콘 톤 통일:** 같은 선 두께·크기.
- **상태색 시스템:** 성공 초록·주의 앰버·위험 빨강 일관.
- *완료 기준:* 사소한 정렬·여백·대비·전환이 정돈돼 "전문적인 제품" 느낌.

## 6. 오류·모순·라이팅 점검(전수) ⭐
- 기능 **모순/중복** 제거: 같은 일을 하는 버튼이 두 곳(예: 가져오기) → 한 곳.
- **용어 통일:** 출석/등원, 학생 관리/학생 명단, 오늘/오늘(일일기록) 등 한 단어로.
- **부제 라이팅:** 개발 메모투("앱에 자체 저장됩니다" 등) → 사용자 말로.
- **빈 상태·안내문** 어색한 문장 다듬기.
- **인센티브:** 수학 외 어디에도 '인센티브' 단어 노출 금지(민감).
- *완료 기준:* 읽었을 때 어색하거나 모순된 문구·버튼이 없다.

## 7. 학생 화면 별도 점검(필요)
- 학생 로그인(예: 민서준) 화면도 위 1~6 기준 동일 적용 — 특히 **포인트 랭킹·시간표·리포트 보기**가 학생 눈높이로 쉽고 깔끔한지.
- 학생 화면에서 **오류 신고(요청)** 진입 한 곳(앞서 만든 창구) 노출.
- *지현님 확인 요청:* 학생 화면에서 특히 불편한 지점 알려주시면 콕 집어 보강.

## 작업 순서
1. **동기화 버튼 통합**(4번) — 지저분함 즉시 해소.
2. **로그인 인사 + 스플래시**(2·1번) — 첫인상.
3. **UI 통일**(3번) — 날짜·카드·버튼·배지.
4. **라이팅·오류·모순 점검**(6번).
5. **고급 디테일**(5번) — 타이포·대비·정렬·전환·스켈레톤.
6. **학생 화면 점검**(7번).
7. 각 단계 **수학 데이터 회귀 확인**.
