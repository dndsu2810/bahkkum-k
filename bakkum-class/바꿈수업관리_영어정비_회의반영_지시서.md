# 바꿈 수업관리 — 영어 정비 + 회의 반영 업데이트 지시서

> 대상: `bakkum-class.dndsu2810.workers.dev` (GitHub: `dndsu2810/bahkkum`)
> 성격: **회의에서 나온 개선·버그 반영 + 영어 모듈을 수학과 통일.**

---

## 0. 이번 업데이트가 다루는 3가지

1. **회의 반영** — 회의에서 나온 개선 요청과 버그를 정리해 손본다. (Part A)
2. **영어 = 수학 통일** — 공통 기능·UI는 수학과 똑같이. 영어 고유 입력만 살려둔다. (Part B)
3. **영어 로드맵 대조** — 기존 영어(구 시스템) 로드맵을 훑어, **이미 있는 건 빼고 없는 것만** 추가한다. (Part C)

> 핵심 정신: **"같은 일은 똑같이. 영어만의 것은 지킨다. 없는 것만 새로 ."**

## A. 회의 반영 — 바로 손볼 것

### ⭐ A-1. 영어 출결 = 수학과 동일 + '보강' 추가

- 현재 영어 출결은 **출석·지각·결석·조퇴** 버튼만 있고, 수학에 있는 **'보강'** 이 없다.
- 영어 출결을 **수학 출결과 똑같은 모양·동작으로** 맞추고, **'보강'을 추가**한다.
- (영어가 수학과 디자인이 다른 이유는 ① 수학을 먼저 만들고 합치면서 어긋났고 ② 영어는 인원이 많아 세로로 길어진 것 — 이번에 수학 기준으로 통일)

### ⭐ A-2. 보강 + 출결 '동시 체크' & 포인트 로직

영어·수학 공통으로 아래 규칙을 적용한다.

- **보강은 켜고/끄는 표시(플래그)** 다. 보강을 켜도 **출석/지각/조퇴/무단결석은 그대로 기록**된다.
  - 즉 "보강 + 출석", "보강 + 지각", "보강 + 조퇴", "보강 + 무단결석" 조합으로 남는다.
- **보강이 켜져 있으면 포인트는 적립되지 않는다.** (출석을 눌러도 포인트 ↑ 안 됨)
- **보강이 꺼져 있으면(평소 수업)** 평소대로 포인트 적립.

> 한 줄: **보강 수업은 출결은 남기되 포인트는 빼고 센다.**

### ⭐ A-3. 포인트 시스템 통일 + 랭킹/시상 `[지현님 값 입력 필요]`

회의 결정사항: **수학도 영어와 같은 '포인트 적립' 방식으로 통일.**

- 수학이 쓰던 별도 방식(키오스크 포인트 직접연동)을 **'포인트 적립' 방식으로 일원화**한다.
- 적립된 포인트는 **키오스크에서 학생이 쓸 수 있게** 연결(추가 사용처는 키오스크에서 풀어줌).
- **포인트 랭킹 탭**에서 **전체 포인트 상위 3명 시상**이 가능하도록 정리.

**[지현님이 채워야 할 것 — 포인트 적립 기준표]**

| 항목 | 적립 점수 |
|------|-----------|
| 출석 | (예: +___) |
| 지각 | (예: +___) |
| 숙제 완료 | (예: +___) |
| 보강 | 적립 없음(0) |
| 무단결석 | (예: 0 / 감점?) |
| … | … |

> 위 표를 정하면 수학·영어에 **똑같이** 적용한다. (값이 정해지기 전엔 적립 로직 골격만 만들고 점수는 비워둠)

### ⭐ A-4. 초등 출결 활성화

- 현재 **초등(노션)에 초등 학생 출결이 진행되지 않고 있다.**
- 초등도 **출결을 무조건 활성화**한다. → **원장님 대시보드가 정상 작동**하려면 필수.
- 초등 출결도 위 A-1·A-2 규칙(보강·포인트)을 동일하게 따른다.

### ⭐ A-5. 학원 일정(캘린더) 버그 3종

회의에서 확인된 버그:

| 증상 | 고칠 내용 |
|------|-----------|
| 기간 일정(예: 24~26일)이 **시작일(24일)에만** 뜨고 종료일까지 안 뜸 | 기간 일정이 **시작~종료 모든 날짜**에 표시되게 |
| 캘린더에서 일정 **삭제가 안 됨** (목록에서만 됨) | 캘린더 화면에서도 **삭제 가능**하게 |
| 노션에서 옮긴/바꾼 일정이 앱에 **반영 안 되거나 지연** | 노션 ↔ 앱 일정 **동기화 반영을 안정화**(지연 최소화) |

### A-6. 자료 / 프린트 배부 기능 (영어에 추가·통일)

회의에서 요청된 핵심 기능. (구 영어 시스템의 '자료 배부 탭'에 해당 — Part C 참고)

- **인쇄할 프린트 목록**에 자료를 등록(예: "호평중3 3과 워크북") → **인쇄 요청** 표시.
- 선생님이 인쇄해두면, 학생에게 **'배부' 클릭 → 그 학생 수업에 들어가고**, **'숙제' 클릭 → 숙제로 들어간다.**
- 자료별로 **누구에게 수업/숙제로 갔는지, 완료됐는지** 한눈에 보이게 연동.
- 수학 쪽에 이미 유사 기능이 있으면 **그것과 같은 모양·동작으로 통일**한다.

## B. 영어 = 수학 UI 통일 원칙

### 똑같이 맞출 것 (공통)
- 출결 버튼/카드, 보강, 포인트 적립, 대시보드 카드, 기록 보기, 자료 배부 등 **두 과목 공통 기능은 디자인·동작을 수학 기준으로 통일.**
- 영어에 **없는 버튼이면 추가**(예: 보강), **다르게 생긴 건 수학 모양으로 교체.**

### 영어만 살려둘 것 (영어 고유 입력)
아래는 영어에 더 들어가는 입력이므로 **없애지 말고 유지**하되, 겉모양(카드·버튼·간격)은 수학 톤과 어울리게 정리:

- 학생이 **오늘 학습목표·숙제를 직접 입력**(항목 추가/삭제/체크)
- **선생님께** 남기는 자유 메모
- **지난 시간 숙제** 합산 표시(학생 입력 + 선생님 사전입력)
- **숙제 검사** 항목별 상태(완료/안함/미흡/미제출)
- 선생님 **사전 입력(preset)** 의 적용 시점(오늘/다음 시간)

> 정리: **틀과 공통 부품은 수학과 같게, 영어만의 입력칸은 그대로 유지.**

## C. 영어 로드맵 대조표 (이미 있는 건 빼고, 없는 것만 추가)

⚠️ **Claude Code는 먼저 실제 코드에서 각 기능이 이미 있는지 확인**한 뒤, **중복은 빼고 빠진 것만** 추가한다. 아래 '추정'은 참고용이며, 코드 확인이 우선이다.

| 로드맵 기능 | 현재 앱 추정 | 조치 |
|-------------|--------------|------|
| 로그인 / 역할 분기(학생·선생님) | 있음 | 유지 |
| 학생 로그인 시 자동 등원 체크 | 불확실 | 확인 후 없으면 추가 검토 |
| 학생 '오늘 입력'(학습목표/숙제 직접입력·체크) | 영어 고유 | **유지**(B 참고) |
| '선생님께' 메모 | 영어 고유 | **유지** |
| 지난 시간 숙제 합산(학생+사전입력) | 불확실 | 확인 후 없으면 추가 |
| 선생님 코멘트 자동 표시 | 있을 가능성 높음 | 유지 |
| 학생 '내 기록 보기'(날짜별) | 있음(기록) | 유지·통일 |
| 대시보드 등원체크 / 학생카드 / 인라인 수정 | 있음(오늘·대시보드) | 유지·통일 |
| 코멘트 입력 — 숙제검사 드롭다운(완료/안함/미흡/미제출) | 영어 고유 가능 | **유지** |
| 사전입력(preset) 적용시점(오늘/다음시간) | 영어 고유 | **유지** |
| **자료 배부 탭**(자료 CRUD·완료현황·배부·배부체크) | 수학에 유사 존재 추정 | **통일 + 영어에 적용**(A-6) |
| 전체 기록 + 리포트(PNG 저장) | 있음(기록·월말리포트) | 유지 |

> 즉, **유지/통일이 대부분**이고, **새로 손볼 핵심은 (1) 영어 출결+보강 통일, (2) 포인트 일원화, (3) 초등 출결, (4) 일정 버그, (5) 자료·프린트 배부** 다섯이다.

## D. 나중에 (확장 메모 — 지금은 만들지 않음)

회의에서 나온 좋은 아이디어지만 이번 범위 밖. **목록만 남겨둔다.**

- **교실 태블릿 출석** + **학부모 번호 입력** + **솔라피/카카오 채널 알림** (어머님 알림용)
- **카카오워크 생일 알림 봇** (오늘 생일자 단체방 자동 공지)
- **교재 신청/구매 관리** (원하는 책 등록 → 구매 여부·구매예정일 표시)
- **마케팅 시기별 키워드** (시험분석·방학특강 등) — 이건 *마케팅 뚝딱 메이커* 쪽 별개 작업

## E. 이미 잘 되는 것 (유지만, 건드리지 말 것)

회의에서 정상 작동 확인됨:

- 학생 **메시지 보내기**(이미 구현) · **공지 배너**(강사 공지) · **오류 접수**
- **시간표 변경 요청**('공통' 카테고리, 승인/거절) · **강사 월별 인센티브 계산** · **원장님 대시보드**

## F. ⚠️ 주의사항

1. **노션 동기화 방향은 건드리지 않는다.** (학생명단 노션→앱, 기록 앱→노션) 일정 동기화는 *안정화*만 하고 규칙은 그대로.
2. **포인트 일원화는 데이터에 영향**이 크다 → 기존 적립 내역이 깨지지 않게 신중히. 적립 기준 점수는 지현님 입력 전까지 비워둔다.
3. 디자인은 **수학 기준 + 기존 공통 부품/토큰** 사용. 새 디자인 만들지 말 것.
4. **한 번에 하나씩**, 화면·데이터 안 깨지는지 확인하며 진행.

## G. 작업 순서

```
1) (확인) 코드에서 영어/수학 현재 기능 대조 → 로드맵 중복 정리 (Part C)
2) 영어 출결을 수학과 통일 + '보강' 버튼 추가 (A-1)
3) 보강 + 출결 동시기록 & 포인트 미적립 로직 (A-2)
4) 포인트 일원화(수학→적립 방식) + 랭킹/시상 골격 (A-3, 점수는 비워둠)
5) 초등 출결 활성화 (A-4)
6) 학원 일정 버그 3종 수정 (A-5)
7) 자료/프린트 배부 통일·영어 적용 (A-6)
8) 영어 고유 입력 유지 점검 + 전체 통일 마무리 (B)
```

## H. Claude Code 시작 명령 (복붙용)

```
첨부한 '영어 정비 + 회의 반영 업데이트 지시서'대로 진행해줘.

먼저 1단계: 실제 코드를 읽고, 영어와 수학이 지금 각각 무슨 기능을 갖고
있는지, 그리고 첨부 로드맵(구 영어 시스템)과 비교해 '이미 있는 것 / 빠진 것'을
표로 정리해서 나한테 보여줘. (Part C 대조표 채우기)

그걸 확인한 뒤 A-1(영어 출결 통일+보강)부터 순서대로 진행하자.

[핵심 규칙]
- 공통 기능은 수학 기준으로 통일. 영어 고유 입력(학습목표·선생님께·숙제검사 등)은 유지.
- 보강이 켜지면 출결은 남기되 포인트는 적립하지 않는다.
- 포인트는 수학·영어 모두 '적립' 방식으로 일원화. 적립 점수는 내가 줄 때까지 비워둘 것.
- 노션 동기화 방향은 절대 건드리지 말 것. 기존 포인트/기록 데이터가 깨지지 않게.
- 디자인은 기존 공통 부품·토큰 사용. 한 번에 하나씩, 깨지는지 확인하며 진행.
```
