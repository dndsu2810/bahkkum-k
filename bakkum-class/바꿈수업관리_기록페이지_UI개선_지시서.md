# 바꿈 수업관리 · 기록 페이지 UI 개선 지시서

> **목적**: 기록(출결·숙제·진도·테스트)이 쌓일수록 화면이 세로로 끝없이 길어지는 문제를 해결한다.
> **절대 원칙**: 기록 데이터는 **하나도 지우거나 줄이지 않는다.** 데이터는 그대로 두고, **보여주는 방식만** 바꾼다.
> **대상 앱**: `bakkum-class.dndsu2810.workers.dev` (Cloudflare Workers + D1 + Notion API)

---

## 1. 현재 문제 진단

네 개의 기록 페이지(출결 기록 / 숙제 기록 / 진도 기록 / 테스트 기록)가 모두 **동일한 구조적 문제**를 갖고 있다.

| 문제 | 설명 |
|------|------|
| **1건 = 가로로 꽉 찬 줄 1개** | 기록 한 건마다 화면 전체 너비의 행 하나를 통째로 차지한다. |
| **무한 누적** | 학생 6명 × 매일 출결/숙제/진도 → 출결 기록은 하루 17줄, 숙제는 하루 12줄까지 생긴다. 이게 날짜별로 끝없이 아래로 쌓인다. |
| **모든 날짜가 펼쳐진 상태** | 과거 날짜까지 전부 펼쳐져 있어서, 오늘 기록 하나 보려고 해도 한참 스크롤해야 한다. |
| **행 높이가 큼** | 한 줄에 담긴 정보(이름·상태·메모)에 비해 세로 공간을 많이 쓴다. 특히 숙제·진도는 교재명이 2줄로 줄바꿈되며 더 길어진다. |

**비유**: 지금은 영수증을 한 장도 안 버리고 둘둘 말아 길게 이어 붙인 **두루마리** 상태다.
이걸 **서랍장**으로 바꾼다 — 날짜별로 서랍을 닫아두고, 보고 싶은 날짜 서랍만 열어보게 한다.

---

## 2. 개선 방향 (4개 페이지 공통 원칙)

### 원칙 A. 날짜 그룹을 "접고 펴는 서랍"으로 만든다 (핵심)
- 현재 날짜별 그룹 헤더(예: `6월 11일 (목)  17건`)를 **클릭하면 접혔다 펴지는** 형태로 바꾼다.
- **기본 상태**:
  - **오늘 + 어제** 날짜 그룹은 펼친 상태로 시작.
  - 그 이전 날짜 그룹은 **모두 접힌 상태**로 시작.
- 접힌 상태일 때 헤더에 **요약 정보**를 함께 보여준다. 예시:
  - 출결: `6월 11일 (목)  ·  17건  ·  출석 15 · 지각 1 · 결석 1`
  - 숙제: `6월 15일 (월)  ·  12건  ·  검사완료 8 · 검사 전 4`
  - 진도: `6월 9일 (화)  ·  2건  ·  진행중 2`
- 헤더 오른쪽에 펼침/접힘 화살표(▸ / ▾) 아이콘 표시.
- 헤더에 "이 날짜 전부 펼치기/접기" 동작이 직관적으로 되도록.

### 원칙 B. 행을 더 촘촘하게 (정보 밀도 ↑)
- 한 행의 세로 높이를 줄여서 같은 화면에 더 많은 기록이 보이게 한다.
- 교재명·단원명이 길어 2줄로 줄바꿈되는 경우, **한 줄로 줄이고 넘치면 말줄임표(…)** 처리. 마우스를 올리면 전체 내용 표시(tooltip).
- 단, 터치(태블릿) 환경을 고려해 행이 너무 빽빽해 누르기 어렵지 않도록 적당한 여백은 유지.

### 원칙 C. 처음엔 최근 것만, 나머지는 "더 보기"
- 페이지를 열었을 때 처음부터 모든 날짜를 그리지 말고, **최근 약 2주(또는 화면 1~2개 분량)**만 먼저 보여준다.
- 맨 아래에 **`+ 이전 기록 더 보기`** 버튼을 두고, 누를 때마다 과거 기록을 이어서 불러온다.
- (이미 출결·숙제는 "2026년 6월" 같은 월 선택 드롭다운이 있다. 이 드롭다운은 유지하되, 선택한 달 안에서도 위 "더 보기" 방식이 동작하도록 한다.)

### 원칙 D. 상단 필터 바를 4개 페이지에 통일
현재 페이지마다 필터가 제각각이다(진도=진행중/전체/완료 탭, 출결·숙제=월 드롭다운, 검색창만 있는 곳도 있음).
아래 **공통 필터 바**를 네 페이지 상단에 동일하게 배치한다.

```
[ 기간 ▾ ]   [ 학생 ▾ ]   [ 상태 ▾ ]            🔍 검색
  └ 최근 7일 / 최근 30일 / 이번 달 / 직접 선택
```

- **기간**: 보이는 날짜 범위를 빠르게 좁힌다. (기본값: 이번 달)
- **학생**: 특정 학생 한 명만 골라 그 학생 기록만 모아 본다.
- **상태**: 페이지 성격에 맞게.
  - 출결 → 출석 / 지각 / 결석 / 보강
  - 숙제 → 검사완료 / 검사 전 / 지연
  - 진도 → 진행중 / 완료
  - 테스트 → (해당 페이지 상태값에 맞게)
- 필터는 **서로 조합 가능**해야 한다. (예: "이번 달 + 김하린 + 결석"만 모아보기)
- 필터가 하나라도 적용된 상태에서는, 위 "원칙 A의 기본 접힘" 대신 **검색 결과를 모두 펼쳐서** 보여준다. (찾는 게 목적이므로)

## 3. 페이지별 세부 지시

### 3-1. 출결 기록
- 상단의 **"오늘 수업 N건" 빠른 입력 카드**(15:00 윤하영 · 출석/지각/결석/더보기 …)는 유지하되, 이 카드 영역은 **접을 수 있게** 한다. 출결 입력이 다 끝난 뒤에는 접어두면 아래 기록 목록이 바로 보이도록.
- 카드를 접으면 한 줄 요약만 표시: `오늘 수업 6건 · 출석 6 / 지각 0 / 결석 0  ▸`
- 그 아래 "출결 기록(80건)" 목록에 위 **원칙 A~D**를 적용한다.

### 3-2. 숙제 기록
- 날짜별 그룹에 **원칙 A~D** 적용.
- 각 행 오른쪽의 **삭제(휴지통) 버튼**은 그대로 둔다. 단, 평소엔 흐리게 두고 마우스 올렸을 때(또는 행 선택 시) 또렷해지게 하여 실수 클릭을 줄인다.
- 교재명("2단원 통합교과형 학습지 발표 준비" 등)이 길어 2줄 되는 부분 → **원칙 B**대로 한 줄 + 말줄임 처리.

### 3-3. 진도 기록
- 기존 **진행중 / 전체 / 완료 탭은 유지**한다(잘 만들어져 있음). 이 탭을 **원칙 D 필터 바의 "상태" 필터와 자연스럽게 합치거나 나란히** 둔다. (중복으로 두 군데서 같은 걸 거르지 않도록 정리.)
- 단원명 2줄 줄바꿈 → 한 줄 + 말줄임.
- 날짜 그룹 접기(**원칙 A**) 적용.
- 상단 "중복 정리" 버튼은 그대로 유지.

### 3-4. 테스트 기록
- 위 세 페이지와 **동일한 구조·원칙(A~D)**을 그대로 적용해 일관성을 맞춘다.

---

## 4. 꼭 지킬 것 (주의사항)

1. **데이터 보존**: 표시 방식만 바꾼다. DB(D1)의 기록 데이터, 삭제/수정 기능, 저장 로직은 절대 건드리지 않는다.
2. **Notion 동기화 규칙 유지**: 기존의 데이터 타입별 단방향 동기화 규칙(학생 명단=Notion→앱, 기록=앱→Notion)을 **그대로 둔다.** 이번 작업은 화면 표시 영역만 다룬다.
3. **디자인 토큰 사용**: 색상·간격·폰트는 **기존 통합 디자인 토큰 시스템**을 그대로 사용한다. 새 색을 임의로 만들지 않는다.
4. **태블릿/모바일 확인**: 학생용 태블릿·강사 휴대폰에서도 접기/펼치기와 필터가 잘 동작하는지 확인한다.
5. **4개 페이지 일관성**: 출결·숙제·진도·테스트가 **똑같은 모양과 동작**을 갖도록 만든다. 한 페이지만 다르게 두지 않는다.

---

## 5. 작업 순서 (권장)

1. 먼저 **숙제 기록 페이지 한 곳**에만 원칙 A(날짜 접기) + B(행 촘촘하게) + C(더 보기)를 적용해 본보기를 만든다.
2. 지현님이 본보기를 확인하고 OK 하면,
3. 같은 구조를 **출결·진도·테스트**에 복제 적용한다.
4. 마지막으로 **원칙 D(공통 필터 바)**를 네 페이지에 일괄 적용한다.

> 한 번에 다 바꾸지 말고, **숙제 기록 한 페이지를 먼저 완성 → 확인 → 나머지 확장** 순서로 진행할 것.

## 6. 완료 기준 (이렇게 되면 성공)

- [ ] 페이지를 열면 **오늘·어제 기록만 펼쳐져** 있고, 과거 날짜는 접혀 있다.
- [ ] 접힌 날짜 헤더만 봐도 그날 요약(건수·상태별 개수)을 알 수 있다.
- [ ] 상단 필터로 **기간·학생·상태**를 조합해 원하는 기록만 모아볼 수 있다.
- [ ] 처음 화면에 모든 기록이 한꺼번에 안 그려지고, "더 보기"로 과거를 불러온다.
- [ ] 네 페이지(출결·숙제·진도·테스트)의 모양과 동작이 똑같다.
- [ ] 기존 기록 데이터·삭제·수정·Notion 동기화는 그대로 잘 작동한다.
